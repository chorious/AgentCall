# AgentCall v2.0 Architecture

AgentCall v2.0 是一个本地多 Agent 控制面，用来让 Codex 指挥 Claude Code 集群协同工作。

核心原则：

```text
Codex owns orchestration.
Claude Code owns bounded execution.
Daemon owns live state.
Report is the handoff contract.
```

## Control Plane

```text
Codex
  -> AgentCall MCP stable bridge
      -> Rust daemon HTTP API
          -> route
          -> PTY session
          -> ACP invocation
          -> hooks ingest
          -> file claims
          -> runtime bindings
          -> session summaries
          -> board
```

MCP 本地只固定暴露 `agentcall_daemon`。业务工具由 daemon 通过 `/api/mcp/tools` 动态提供，并通过 `/api/mcp/call` 执行。

## Route-First Workflow

Codex 默认工作流：

```text
board -> route -> session/report
```

- `agentcall_board`：低成本看全局状态和 attention。
- `agentcall_route`：唯一高层派工入口，支持 `runtime=auto|pty|acp`。
- `agentcall_session`：读取 compact summary 或 clean tail。
- `agentcall_session_send`：对 PTY worker 发送输入、continue、stop、report request。
- `agentcall_report`：请求或接受 report。

`agentcall_delegate` 不再作为默认路径；route 是统一入口。

## Runtime Choices

### PTY

PTY 是人类可视化和 handoff 路线。daemon 启动 Claude Code PTY session，并注入：

```text
AGENTCALL_WRAPPER_SESSION=<session name>
```

Claude Code 的 cwd 不由 route 请求决定，而由 daemon local config 的 `claude_workspace` 强制决定。

### ACP

ACP 是 bounded agents-as-tools 路线。daemon 拥有 ACP child process，负责 timeout、stdout/stderr 读取、JSON-RPC request id、permission response 和结果落盘。

v2.0 中 Rust native ACP client 已用 Python reference 做同进同出 parity 测试。

## Daemon Single-Writer

daemon 是 live 共享状态的唯一写者：

- events
- file claims
- active sessions
- runtime bindings
- route state
- session summary
- board projection

Python 保留为 hooks installer、测试辅助、legacy/debug 胶水，不作为 live state writer。

## Hooks And Binding

Claude/Codex hooks POST 到 daemon `/api/hooks/ingest`。

绑定主键：

```text
AGENTCALL_WRAPPER_SESSION
```

如果 hook 没有 wrapper env 且没有既有反查记录，daemon 标记为 `unbound`，不会使用 cwd、PID、窗口标题或启动顺序猜测归属。

状态来源优先级：

```text
daemon lifecycle / hook events
  > report and file validators
  > TUI extractor hints
```

## File Claims

只有写工具会 claim：

```text
Write / Edit / MultiEdit / NotebookEdit
```

读工具只 observe：

```text
Read / Glob / Grep
```

同一文件的 active write claim 冲突会被 daemon 稳定拒绝。

## Local Config

提交模板：

```text
config/agentcall.example.json
```

本机配置：

```text
config/agentcall.local.json
```

`claude_workspace` 是 Claude Code PTY 的强制 cwd，也是 hooks 绑定语义的一部分。缺失时 daemon health 返回 `status=config_missing`，Claude PTY route 拒绝启动。

`acp_command` 是 daemon-owned ACP adapter 命令。ACP route 的命令解析顺序是：

```text
route.adapter_command
  -> config/agentcall.local.json acp_command
  -> AGENTCALL_ACP_COMMAND
```

因此调用方按协议传 `runtime=acp` 即可使用本机默认 ACP adapter；只有需要临时覆盖 adapter 时才传 `adapter_command`。

## Output Model

每个 PTY session 维护三层输出：

- raw output：完整 PTY byte/debug 流。
- clean output：UTF-8 streaming decoder + ANSI 清理后的可读输出。
- llm summary：Codex 默认读取的紧凑状态。

Codex 默认不读 raw terminal；只有 debug 或低置信度时才展开 clean/raw 输出。

## Non-Goals

- 不自动 kill 用户可见 PTY worker。
- 不实现 Claude Code 生命周期自动销毁或自动续接。
- 不用 Python 作为 live state writer。
- 不用 TUI grep 覆盖 hook/report/daemon 结构化状态。
- 不把本机 `D:\guKimi` 写成发布硬编码默认。
