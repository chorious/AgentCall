# AgentCall v6.8 Code Plan: Owner-scoped Batch Snapshot

## Summary

v6.8 解决 Codex 侧 `agentcall_session` 体感慢和 batch state 所有权风险。结论来自 `docs/reports/report_session_latency_batch_state_2026-06-14.md` 与 GPT Pro 反馈：慢点主要在 Codex-hosted MCP tool-call envelope / UI / 调度层，而不是 Rust daemon 或 `agentcall-mcp` handler。优化方向不是让 Codex 并发调用多个 `agentcall_session`，而是让 Codex 通过一次 owner-scoped `agentcall_board` 拿到所有当前 worker 的 compact 状态。

v6.8 不是单纯性能 patch，而是状态面的产品收口：

- 默认状态查询必须 pure-read。
- batch state 必须 owner-scoped。
- control token 必须 lazy mint。
- `scope=all` 必须 debug/read-only。
- prompt gate 推进不得由 summary/board 查询顺手触发。

## Non-goals

- 不新增 `agentcall_sessions` MCP 工具。
- 不支持 `agentcall_session(name="*")`。
- 不把 `scope=all` 作为 Codex 默认状态入口。
- 不在 batch board 中返回 control token。
- 不在默认 summary / board 查询中 patch route、append event、submit prompt、mint token。
- 不让 session name 成为 authority；session name 只是索引。
- 不在无 owner 绑定时默默 fallback 成可控制的 `codex`。
- 不把 raw terminal、events、tool outputs、clean tail 放进默认 batch response。
- 不修改冻结计划 `docs/v6.2-code-plan.md`。

## Current Evidence

三层延迟实验：

| Layer | Call | Observed |
| --- | --- | ---: |
| daemon HTTP direct | `agentcall_board(compact/all)` | first `46.88ms`, then `2.35-3.80ms` |
| standalone MCP stdio | `agentcall_board(compact/all)` | avg `2.48ms`, max `3.64ms` |
| Codex built-in MCP tool | `agentcall_board(compact/all)` | `6.1ms` |
| Codex built-in MCP tool | `agentcall_session(route-pty-1803903)` | `12.99s` |
| same MCP server timing log | same `agentcall_session` | `call_ms=171ms`, `render_ms=0`, `2507 bytes` |

The precise bottleneck is:

```text
Codex-hosted MCP tool-call envelope / scheduling / UI, not daemon handler latency.
```

Therefore v6.8 reduces tool-call count and response weight.

## Target User Flow

Default multi-worker state inspection:

```text
agentcall_board(view=compact, filter=all, section=sessions, scope=mine)
```

Only inspect one session when needed:

```text
agentcall_session(name=..., include=["summary"])
agentcall_session(name=..., include=["summary", "control"])  # only before destructive/phase-changing action
```

Only use debug/raw when explicitly needed:

```text
agentcall_session(name=..., view=events|debug|raw)
```

No new MCP tool.

### `scope=all`

`scope=all` is debug/read-only. It must not return control tokens or executable destructive action args.

Each out-of-owner worker must carry:

```json
{
  "owner_visible": false,
  "readonly_debug_projection": true,
  "control": {
    "available": false,
    "token_included": false,
    "reason": "scope_all_debug_readonly"
  }
}
```

### `agentcall_session`

Default summary must be pure-read and no-token.

Token-bearing view must be explicit:

```json
{
  "include": ["summary", "control"]
}
```

If caller owner is missing or mismatched, token minting returns structured `owner_unbound` or `owner_mismatch`.

## Owner Model

### Owner derivation

MCP bridge derives owner per request:

1. `AGENTCALL_OWNER_ID`
2. `CODEX_THREAD_ID`, normalized as `codex-thread-<id>`
3. explicit local legacy policy, only if configured

No silent controllable fallback to `codex`.

### Owner context in daemon MCP calls

`/api/mcp/call` already accepts:

```json
{
  "client": {
    "owner_id": "..."
  }
}
```

v6.8 must thread this owner through:

- route start
- board `scope=mine`
- session summary control availability
- session send validation
- report accept validation
- prompt gate auto-submit

### Unbound owner behavior

Read-only calls may return `owner_unbound` metadata.

Control/write calls must fail:

```json
{
  "ok": false,
  "status": "owner_unbound",
  "error": {"code": "owner_unbound"},
  "reason": "control actions require AGENTCALL_OWNER_ID or CODEX_THREAD_ID",
  "next_step": "bind owner and retry"
}
```

## Code Changes

### P0: Fix owner hardcoding before batch default

#### `routes.rs`

Current risk: route owner lease may use the caller owner, but first prompt submission still uses hardcoded `"codex"` in prompt submission helpers.

Change:

```rust
submit_pty_prompt_with_ack(state, route_id, session_name, prompt, handoff, owner_id)
submit_pty_prompt_without_hook_ack(state, route_id, session_name, prompt, owner_id)
```

Remove hardcoded:

```rust
"owner_id": "codex"
```

Use actual route owner.

#### `prompt_gate.rs`

Current risk: daemon auto-submit uses `"owner_id": "codex"`.

Change:

```rust
fn daemon_auto_submit_pending_prompt(...)
```

must resolve owner from the session owner lease:

```rust
owner_id_for_session(state, wrapper_session) -> Result<String, String>
```

Then pass the real owner id into `prepare_session_send_command`.

#### `control.rs`

Current risk:

```rust
control_summary_for_session(state, session_id, None)
```

defaults to `"codex"`.

Change default summary behavior:

- no owner + no explicit control request => return no-token metadata;
- explicit control request + no owner => return `owner_unbound`;
- owner mismatch => return `owner_mismatch`.

#### `mcp.rs`

Thread `McpClientContext.owner_id` into:

- `mcp_board`
- `mcp_session`
- `mcp_session_send`
- `mcp_report`

Do not only thread it into `mcp_route`.

### P0: Pure-read worker snapshot

Add a pure-read snapshot path:

```rust
pub(crate) struct WorkerSnapshotView { ... }

pub(crate) fn worker_snapshot_for_session(
    state: &AppState,
    session_name: &str,
    owner_id: Option<&str>,
) -> WorkerSnapshotView
```

Allowed reads:

- session projection from `state.projections` / store projection;
- in-memory session status;
- compact route fields;
- compact report fields;
- owner lease metadata;
- workspace projection.

Forbidden calls:

- `refresh_prompt_gate_timeouts_for_session`
- `patch_route_record`
- `append_agent_event`
- `complete_command_event`
- `prepare_session_send_command`
- `submit_session_command`
- `control_summary_for_session` token minting

### P0: Compact board uses snapshots

Change:

```rust
v6_compact_board_state(...)
```

from:

```rust
worker_state_for_session(state, &session.name).to_board_worker()
```

to:

```rust
worker_snapshot_for_session(state, &session.name, owner_id).to_board_worker()
```

Apply owner filter before response:

```rust
scope=mine => only caller owner
scope=all  => debug readonly projection
```

### P1: Session summary default uses snapshot

Change `session_summary_view` default path to use `worker_snapshot_for_session`.

Only use full `worker_state_for_session` when:

- explicit debug refresh is requested;
- phase-changing action is being prepared;
- `include` contains a future explicit `"refresh"` marker.

### P1: Lazy control token

Default:

```json
"control": {
  "available": true,
  "token_included": false,
  "required_for": [...]
}
```

Explicit:

```text
agentcall_session(include=["summary", "control"])
```

returns a fresh short-lived token only if caller owner matches the owner lease.

### P1: Background prompt gate maintenance

Move prompt gate timeout maintenance out of default query paths.

Acceptable drivers:

- route start follow-up maintenance;
- daemon periodic maintenance;
- explicit control/debug refresh.

Default board/session summary must not trigger prompt auto-submit.

## Response Budget

Hard targets:

| Response | Target | Hard cap |
| --- | ---: | ---: |
| `agentcall_board compact sessions` | <= 6KB | 8KB |
| `agentcall_session summary` | <= 2KB | 4KB |
| `agentcall_session events compact` | <= 16KB | 32KB |
| debug/raw | explicit only | must include budget metadata |

Default compact worker fields:

- `name`
- `owner_visible`
- `state`
- `why`
- `can_wait`
- `primary_action`
- `attention_status`
- `pending_interaction.kind`
- `last_progress_age_seconds`
- `report.ready/status/path`
- `control.available/token_included/required_for`
- minimal `workspace.target_workspace` only when useful

Do not include by default:

- control token
- full route result
- full prompt gate object
- events
- terminal screen
- clean tail
- tool output
- raw hook payload
- full containment object

## Tests

### Ownership tests

- owner A can start up to 6 workers even when owner B has 6 workers.
- owner A `board(scope=mine)` does not show owner B workers.
- owner A cannot `stop/send/interrupt/request_report/accept` owner B session.
- `board(scope=all)` returns owner B workers as readonly debug projections only.
- `board(scope=all)` never returns control token.
- no owner id + control action returns `owner_unbound`.
- no owner id + readonly board returns `owner_unbound` metadata, not hidden control.

### Prompt gate tests

- route first prompt uses route owner id, not `"codex"`.
- prompt gate auto-submit uses session owner id.
- default `agentcall_session(summary)` does not call `refresh_prompt_gate_timeouts_for_session`.
- default `agentcall_board(compact)` does not patch route or append event.

### Performance tests

- 6 live worker compact board daemon-side <= 300ms.
- compact board response <= 8KB.
- standalone MCP stdio board call remains <= 100ms in local test.
- repeated board calls do not grow events or route state.

### Regression

```powershell
cargo test --workspace
python -m pytest -q
python agentcall.py release-check
```

## Acceptance Criteria

- `agentcall_board(view=compact, filter=all, section=sessions, scope=mine)` is the default multi-worker status path.
- The default board returns all current-owner worker states in one call.
- Default board and session summary are pure-read and have no side effects.
- Default board/session summary do not mint or return control tokens.
- `scope=all` is debug/read-only and cannot be used as a control surface.
- Owner id is threaded through board/session/session_send/report, not only route.
- Non-owner control/report actions fail with structured enum-backed errors.
- 6-worker board batch meets the latency and response budget targets.
- AGENTS.md and the AgentCall supervisor skill instruct Codex to use board batch before single-session inspection.

## Implementation Order

1. Add owner context plumbing to all daemon MCP handlers.
2. Fix route prompt and prompt gate hardcoded `"codex"` owner.
3. Add owner filter helpers and owner-visible metadata.
4. Implement `worker_snapshot_for_session`.
5. Switch compact board to snapshots and owner filtering.
6. Make default session summary no-token and pure-read.
7. Add explicit control include behavior.
8. Add tests for ownership, no side effects, response budget, and performance.
9. Update AGENTS.md / supervisor skill / README snippets.
10. Run release validation and restart daemon.

## Open Questions

- Should unbound readonly `scope=mine` return an empty worker list or an explicit `owner_unbound` warning with empty list? Recommended: warning + empty list.
- Should daemon expose a periodic maintenance loop for prompt gate, or should maintenance run on route start and explicit debug refresh only? Recommended: start with route-start maintenance plus explicit refresh; add periodic loop only if evidence requires it.
- Should `scope=all` require an explicit debug flag beyond `scope=all`? Recommended: yes if accidental use continues.
