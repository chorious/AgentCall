# AgentCall v0.5 实现说明

v0.5 在 v0.4 的 MCP/control plane 基础上，把 Claude Code handoff 方向推进到
可安装、可观测、可冲突防护、可前端查看。

## 真实 Claude Code Hooks

新增安装脚本：

```powershell
.\scripts\install-claude-hooks.ps1 -Root E:\Project\AgentCall
```

默认写入：

```text
.claude/settings.local.json
```

安装内容包括：

```text
SessionStart
UserPromptSubmit
PreToolUse
PostToolUse
Notification
Stop
SubagentStop
PreCompact
SessionEnd
```

每个 hook 都调用：

```text
scripts/agentcall-claude-hook.py
```

这个 handler 会从 stdin 读取 Claude Code hook JSON，调用
`python -m agentcall hook ingest ...`，再把结果返回给 Claude Code。

`SessionStart` 和 `UserPromptSubmit` 会返回：

```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "..."
  }
}
```

`PreToolUse` 如果发现文件 claim 冲突，会返回：

```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "file claim conflict"
  }
}
```

## File Claim / Conflict Policy

新增 `FileClaimPolicy`：

- `Write`
- `Edit`
- `MultiEdit`
- `NotebookEdit`

这些写入类工具在 `PreToolUse` 时会尝试 claim 文件。

如果另一个 active session 已经 claim 了同一个文件，AgentCall 会 deny 当前工具调用。
`PostToolUse` 会记录实际写入观察，`Stop` / `SubagentStop` / `SessionEnd` 会释放该
session 的 active claims。

状态写入：

```text
.agentcall/state/file_claims.json
```

查询：

```powershell
python -m agentcall claims --json
```

## Transcript 索引和摘要

新增：

```powershell
python -m agentcall transcript index <transcript.jsonl> --session-id <id>
python -m agentcall transcript list
```

索引结果写入：

```text
.agentcall/state/transcripts.json
```

摘要字段包括：

- messages
- user_messages
- assistant_messages
- tool_uses
- tool_results
- last_text

如果 hook payload 带 `transcript_path`，hook ingestion 会自动尝试索引。

## 更强 Router

`agentcall route` 新增信号：

```powershell
--risk
--phase
--expected-minutes
--parallel-children
```

返回值新增：

- confidence
- alternatives

这样 Codex 不只是得到 runtime 建议，也能看到何时升级/降级到另一条路径。

## 状态 Board 前端

daemon 新增页面：

```text
http://localhost:3293/board
```

新增 API：

```text
GET /api/board
```

页面展示：

- Claude Code sessions
- file claims
- reports
- transcripts
- recent events
- metrics

这不是营销页面，是工程状态面板。终端四宫格仍然保留在 `/`。

## MCP v0.5

MCP server 版本升级到 `0.5.0`，新增或增强：

```text
agentcall_file_claims
agentcall_transcript_index
agentcall_transcripts_list
agentcall_board
agentcall_route_task
```

`agentcall_route_task` 支持 v0.5 router 的新信号。

- MCP file claims 和 transcript index/list 工具走通。

## 当前边界

v0.5 已经有真实 hook 安装脚本和 handler，但还没有在用户当前正在运行的 Claude Code
窗口里自动执行安装。安装后需要在 Claude Code 中用 `/hooks` 检查 loaded hooks。

board 是可用状态前端，但还不是最终产品 UI。下一步应该评估效果：看它能否降低
Codex/用户判断“谁在做什么、谁卡住、哪里冲突、上下文是否足够”的成本。
