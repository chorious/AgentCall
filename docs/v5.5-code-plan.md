# AgentCall v5.5 Plan：Daemon-Minted Control Safety

## Summary

v5.5 修正 v5.4 safety-check 的接口过度问题。安全目标继续保留，但不再要求 Codex 手动维护 daemon 内部字段，例如 `owner_lease_id`、`lease_generation`、`projection_last_session_seq`、`idempotency_key`。

核心方向：

> Codex 发 intent；daemon 负责 mint command envelope、attach current lease、dedupe、校验 control epoch、返回结构化结果。

v5.5 是“控制接口减负 + daemon 内部事务权威”，不是再发明一套复杂 key 系统。实现必须保持简单、短命、服务端权威。

## Relationship To v5.4.x

- v5.5 **supersedes** v5.4.2 的 caller-supplied destructive precondition 接口，面向正常 MCP caller 时不再推荐手动传 `owner_lease_id` / `lease_generation` / `projection_last_session_seq`。
- v5.4.2 旧 `precondition` 路径只保留一个版本作为 debug compatibility。
- v5.5 不替代 v5.4.1 lifecycle 修复；route short transaction、stop/kill 分离、lease 只在 observed process exit 后释放、actor priority/panic guard 仍必须保留。
- v5.5 不削弱 v5.4/v5.4.3 的 path containment、binding fail-closed、file claim、daemon token、projection-first、MCP output budget。

## Problems To Fix

### P0：接口层暴露内部 lease 细节

当前 `agentcall_session_send` schema 暴露：

- `owner_lease_id`
- `lease_generation`
- `precondition`
- `idempotency_key`

这些字段本质是 daemon 内部事务信息。LLM supervisor 跨 turn、跨 tool call、跨上下文压缩后很难可靠维护它们。

### P0：route prompt command 绕过 lease attach

真实样本显示：

```text
pty.session_started:
  owner_lease_id = lease-...
  lease_generation = 1

command.accepted for route prompt:
  owner_lease_id = unleased
  lease_generation = 0
```

route-created prompt dispatch 必须统一走 active route lease 的 command envelope construction。这个不能靠 Codex 补 key。

### P0：projection seq 粒度太细

`projection_last_session_seq` 对普通 hook、command、progress 都敏感。普通 `PostToolUse` 或 `command.completed` 不应该让 destructive control 立刻 stale。

真正应该让控制失效的是高风险状态变化，例如 permission prompt、policy block、report_ready、route phase、terminal state、owner lease 变化。

因此 v5.5 引入 `control_epoch`，只作为高风险动作 freshness gate。

### P1：普通 send 与 destructive action 混在同一强安全层

`send` / `continue` / `request_report` 是 benign control；daemon 应自动 attach lease、自动 idempotency、必要时 queue。

`interrupt` / `stop` / `kill` / `approve_plan` / `start_auto` 是 destructive 或 phase-changing control，才需要 fresh control token。

## Non-goals

- 不降低 path containment、file claim、binding fail-closed 的强度。
- 不允许 unbound worker 执行写入或非只读 bash。
- 不把 `report/accept` 设计成 lease release。
- 不用 Python 实现 live control/session/projection writer。
- 不恢复 ACP/SDK 主线。
- 不扩大 MCP 工具数量。
- 不把 `daemon_token`、`control_token`、`idempotency_key` 混成同一种东西。
- 不从 `daemon_token` 派生 `control_token`。
- 不要求 Codex 继续手动拼 `owner_lease_id` / `lease_generation`。
- 不让普通 progress event 推进 destructive control freshness gate。
- 不在 board 返回完整 `control_token`。
- 不在 mint token 时隐式创建或 reacquire lease。

## Target Model

### Action Levels

```text
Level 0 observe
  board / session / events / report reads.
  No control token.

Level 1 benign control
  send / continue / request_report / revise_plan.
  Daemon auto attaches current owner lease.
  Daemon auto generates idempotency key.
  If worker is busy, daemon queues supervisor instruction.

Level 2 menu or phase control
  select_option / approve_plan / start_auto.
  Requires fresh control_token unless daemon can prove a current structured interaction allows the action.

Level 3 destructive control
  interrupt / stop / kill.
  Requires fresh control_token.
```

### Token Boundaries

```text
daemon_token
  Protects local HTTP API.
  Config/env level.

control_token
  Protects high-risk action against one session.
  Short-lived, daemon-minted, server-side opaque handle.

idempotency_key
  Prevents retry duplicate side effects.
  Daemon-generated by default.
```

These must remain separate.

### Session Summary Control Object

`agentcall_session(view=summary, name=...)` returns the full control object:

```json
{
  "session": "worker-a",
  "projection_seq": 123,
  "control_epoch": 7,
  "control": {
    "token": "ctl_opaque_...",
    "owner": "codex",
    "expires_in_seconds": 60,
    "allowed_actions": [
      "send",
      "continue",
      "request_report",
      "interrupt",
      "stop"
    ],
    "token_required_for": [
      "interrupt",
      "stop",
      "kill",
      "approve_plan",
      "start_auto"
    ]
  }
}
```

Compact board must **not** return full token. Board may return only:

```json
{
  "session": "worker-a",
  "attention": "needs_permission",
  "control_epoch": 7,
  "control_available": true,
  "allowed_actions": ["request_report", "interrupt", "stop"],
  "token_required_for": ["interrupt", "stop"]
}
```

Codex reads single-session summary before high-risk action:

```json
{
  "name": "worker-a",
  "action": "stop",
  "control_token": "ctl_opaque_..."
}
```

Codex should not need:

```json
{
  "owner_lease_id": "...",
  "lease_generation": 1,
  "precondition": {
    "projection_last_session_seq": 123
  }
}
```

## Implementation Plan

### Step 0 / P0：Fix Route Prompt Envelope First

Update:

```text
crates/agentcall-daemon/src/routes.rs
crates/agentcall-daemon/src/commands.rs
```

Route prompt dispatch must:

- install route owner/workspace leases first。
- construct command envelope from the active route owner lease。
- use fixed idempotency key:

```text
route_prompt:{route_id}:{session_name}
```

- dedupe retry instead of typing prompt twice。
- never emit:

```text
owner_lease_id = unleased
lease_generation = 0
```

This is independent of the full control token system and must land first or with the first v5.5 implementation slice.

### Step 1 / P0：Introduce Control Epoch

Update:

```text
crates/agentcall-daemon/src/projection.rs
crates/agentcall-daemon/src/events.rs
```

Add fields:

```rust
control_epoch: u64
control_epoch_reason: Option<String>
control_epoch_updated_at: Option<String>
```

Increment `control_epoch` only on material high-risk transitions:

- owner lease released / expired / reassigned。
- session ended / failed / actor failed / writer failed。
- permission prompt appears, changes, or clears。
- policy block first appears in current blocker group。
- policy block materially changes target/reason category。
- policy block clears。
- report_ready changes from false to true。
- route phase changes。
- prompt ack missing。
- binding trust changes。

Do not increment `control_epoch` for:

- normal progress。
- benign `PostToolUse`。
- repeated same policy denial count increase。
- queued instruction `command.completed`。
- clean terminal updates。

### Step 2 / P0：Add Server-Side Control Token Module

Add:

```text
crates/agentcall-daemon/src/control.rs
```

Token implementation for v5.5 MVP:

```text
format:
  ctl_<base64url(32 random bytes)>

entropy:
  32 CSPRNG bytes.

storage:
  in-memory HashMap<token_hash, ControlTokenClaims>.

restart behavior:
  all old tokens invalid.
  return expired_control_token or daemon_restarted_refresh_summary with refresh guidance.

logging:
  never log raw token.
  if necessary, log token_hash prefix only.
```

Do **not** implement signed payload in v5.5 MVP unless crypto dependencies are added deliberately. Do **not** derive control tokens from `daemon_token`.

Types:

```rust
pub(crate) struct ControlSnapshot {
    pub(crate) session_id: String,
    pub(crate) owner_id: String,
    pub(crate) control_epoch: u64,
    pub(crate) projection_seq: u64,
    pub(crate) expires_at_ms: u64,
    pub(crate) allowed_actions: Vec<String>,
    pub(crate) token: Option<String>,
}

pub(crate) struct ControlTokenClaims {
    pub(crate) session_id: String,
    pub(crate) owner_id: String,
    pub(crate) owner_lease_id: String,
    pub(crate) lease_generation: u64,
    pub(crate) control_epoch: u64,
    pub(crate) expires_at_ms: u64,
    pub(crate) allowed_actions: Vec<String>,
    pub(crate) route_id: Option<String>,
    pub(crate) phase: Option<String>,
    pub(crate) plan_artifact_hash: Option<String>,
    pub(crate) pending_interaction_id: Option<String>,
}
```

Functions:

```rust
mint_control_token(state, session_id, owner_id) -> Result<ControlSnapshot, ControlError>
validate_control_token(state, session_id, action, token) -> Result<ControlTokenClaims, ControlError>
```

Minting rules:

- requires live session。
- requires active owner lease。
- requires owner_id matches active lease。
- requires session is not terminal。
- must not call `ensure_owner_lease`。
- must not create/reacquire lease implicitly。
- returns `control_available=false` style summary if unavailable。

### Step 3 / P0：Atomic Validation And Dispatch

Update:

```text
crates/agentcall-daemon/src/commands.rs
crates/agentcall-daemon/src/actor.rs
```

High-risk action path must handle race between token validation and execution.

Required behavior:

- MCP/session_send validates token and registers command in one daemon critical section where possible。
- `CommandEnvelopeV1` carries control claims snapshot:

```rust
control_epoch: Option<u64>
control_token_hash: Option<String>
```

- Actor revalidates current session/lease/control_epoch immediately before destructive effect。
- If epoch/lease/session changed before actor execution, reject or no-op safely and emit structured event。

This double-check is required for:

- `interrupt`
- `stop`
- `kill`
- `approve_plan`
- `start_auto`

### Step 4 / P0：Refactor Session Send Safety

Update:

```text
crates/agentcall-daemon/src/commands.rs
crates/agentcall-daemon/src/mcp.rs
crates/agentcall-mcp/src/tools.rs
```

New recommended request shape:

```json
{
  "name": "worker-a",
  "action": "request_report",
  "text": "Write concise report."
}
```

For Level 1:

- daemon attaches active owner lease。
- daemon generates idempotency key if omitted。
- no caller-supplied owner lease fields required。
- if busy and no attention, queue supervisor instruction。
- if permission prompt, return `blocked_by_permission_prompt`。
- if policy block, return `blocked_by_policy`。

For Level 2/3:

```json
{
  "name": "worker-a",
  "action": "interrupt",
  "control_token": "ctl_..."
}
```

Legacy compatibility:

- Keep accepting `precondition` / `owner_lease_id` / `lease_generation` for one version as debug-only low-level path。
- If both `control_token` and old fields exist, `control_token` wins。
- Normal MCP schemas and docs must not encourage callers to fill old lease fields。

### Step 5 / P1：Action-Specific Idempotency

Default idempotency is daemon-owned, but action-specific:

```text
route prompt:
  route_prompt:{route_id}:{session_name}

continue:
  session:{name}:continue:epoch:{control_epoch}

request_report:
  session:{name}:request_report:epoch:{control_epoch}

revise_plan:
  session:{name}:revise_plan:epoch:{control_epoch}:hash:{text}

send with explicit text:
  prefer mcp_request_id when available.
  if no mcp_request_id, generate a unique key plus short retry-window payload hash.
  do not permanently dedupe identical natural-language text across distinct user turns.
```

MCP bridge should forward JSON-RPC request id or synthesize `mcp_request_id` for daemon idempotency:

```json
{
  "mcp_request_id": "jsonrpc-call-id",
  "name": "worker-a",
  "action": "send",
  "text": "..."
}
```

### Step 6 / P1：Structured Control Refusals

Expected control refusals are business states, not transport errors.

For expected refusal:

```text
MCP tool call succeeds.
Payload: {"ok": false, "status": "...", "next_step": "..."}
```

For parse/internal/daemon crash:

```text
MCP tool call returns tool error.
```

Expected refusal statuses:

- `missing_control_token`
- `stale_control_token`
- `expired_control_token`
- `daemon_restarted_refresh_summary`
- `action_not_allowed`
- `owner_conflict`
- `session_not_live`
- `prompt_not_submitted`
- `blocked_by_policy`
- `blocked_by_permission_prompt`

Example:

```json
{
  "ok": false,
  "status": "stale_control_token",
  "reason": "session state changed after supervisor observation",
  "current": {
    "attention": "needs_permission",
    "control_epoch": 8
  },
  "next_step": "refresh agentcall_session(view=summary) and retry with control.token"
}
```

### Step 7 / P1：Permission Interaction Identity

Structured permission/menu state should include:

```json
{
  "pending_interaction": {
    "kind": "permission_menu",
    "interaction_id": "perm-...",
    "control_epoch": 8,
    "options": [
      {"index": 1, "semantic": "approve"},
      {"index": 2, "semantic": "inspect"},
      {"index": 3, "semantic": "deny"}
    ]
  }
}
```

`select_option` may be allowed with current `interaction_id` if daemon can prove the menu is current. Otherwise use `control_token`.

### Step 8 / P1：Plan Approval Binding

`approve_plan` / `start_auto` tokens must bind the plan object:

- route_id。
- phase = plan。
- plan_artifact_hash if available。
- pending_interaction_id if approval came from an interactive plan prompt。

If the plan changes after token minting, old approval token is stale.

### Step 9 / P1：Prompt Not Submitted State

Update:

```text
crates/agentcall-daemon/src/routes.rs
crates/agentcall-daemon/src/summary.rs
crates/agentcall-daemon/src/projection.rs
```

If route status is `started_pending_prompt_ack` beyond prompt ack deadline:

- set `attention_status=prompt_not_submitted`
- set `needs_attention=true`
- increment `control_epoch`
- recommended action:

```text
submit_prompt_or_restart_worker
```

Do not show healthy `working/none`.

Update:

Recommended `agentcall_session_send` fields:

- `name`
- `action`
- `text`
- `enter`
- `control_token`
- `idempotency_key` optional/debug
- `mcp_request_id` internal/optional

Deprecated low-level fields:

- `owner_id`

Docs must say:

## Test Plan

### Route Prompt

- route prompt command uses active route lease。
- route prompt idempotency key is route/session bound。
- route prompt retry dedupes and does not type a second prompt。
- route prompt never records `unleased` owner lease。
- route prompt can be dispatched once even when hook ack is delayed。

### Control Token

- summary mints token for live owned session。
- token is not minted for terminal session。
- token is not minted for orphan projection without live session。
- token validates for allowed destructive action。
- token rejects wrong session。
- token rejects wrong owner。
- token rejects stale control_epoch。
- token rejects expired token。
- token rejects action not in allowed_actions。
- daemon restart invalidates in-memory token and returns refresh hint。
- raw control token never appears in events/logs。

### Atomicity / Race

- token validated then epoch changes before actor execute is rejected or no-ops safely。
- token validated then owner lease released before actor execute is rejected。
- token validated then session exits before actor execute no-ops terminal。

### Control Epoch

- permission prompt increments control_epoch。
- permission menu option change increments control_epoch。
- policy block first appearance increments control_epoch once。
- same policy block repeat does not bump every time。
- policy block cleared increments control_epoch。
- report_ready false to true increments control_epoch once。
- session end increments control_epoch。
- benign progress does not increment control_epoch。
- command.completed for queued instruction does not increment control_epoch。

### Benign Session Send

- `continue` without token/idempotency succeeds。
- repeated same `continue` in same control epoch dedupes。
- `request_report` without token queues when worker busy。
- `send(text)` without idempotency uses `mcp_request_id` when available。
- identical `send(text)` from distinct request ids is not permanently deduped。
- benign send owner conflict returns structured `owner_conflict`。
- benign send permission prompt returns `blocked_by_permission_prompt` with `ok=false`。
- benign send policy block returns `blocked_by_policy` with `ok=false` and is not queued。

### Destructive / Phase Control

- `interrupt` without token returns `ok=false/status=missing_control_token`。
- `interrupt` with fresh token dispatches。
- `stop` with stale token returns structured stale token response。
- `approve_plan` token binds route_id + phase + plan_artifact_hash。
- revised plan invalidates old approval token。
- old precondition path still works in debug compatibility tests。

### Prompt Ack

- route pending prompt beyond deadline projects `prompt_not_submitted` attention。
- prompt_not_submitted increments control_epoch。
- Codex summary no longer reports healthy working when prompt was never submitted。

### Integration Smoke

1. Start daemon。
2. Route one PTY worker。
3. Inspect `agentcall_session(view=summary)`。
4. Confirm control object exists。
5. Confirm board does not include full token。
6. Call `session_send(action=continue)` without idempotency key。
7. Confirm no 400。
8. Call `session_send(action=interrupt)` without token。
9. Confirm tool-success payload with `ok=false/status=missing_control_token`。
10. Call `session_send(action=interrupt, control_token=...)`。
11. Confirm accepted or safe structured refusal based on live state。

## Implementation Order

1. Fix route prompt envelope and route prompt idempotency。
2. Add `control_epoch` fields and bump rules, but do not mint token yet。
3. Add server-side in-memory control token table and single-session summary token output。
4. Route destructive actions through token validation plus actor revalidation。
5. Refactor Level 1 benign control auto lease/idempotency action by action。
6. Add permission `pending_interaction` identity。
7. Add plan approval binding。
8. Clean MCP schema/docs/skills。

## Acceptance Criteria

- Codex no longer needs to manually supply `owner_lease_id` / `lease_generation` for normal MCP session control。
- `idempotency_key` is optional for normal MCP usage and daemon-generated by default。
- Board never returns full `control_token`。
- Single-session summary returns short-lived server-side opaque token when live owned control is available。
- `mint_control_token` never creates/reacquires lease implicitly。
- Destructive actions require daemon-minted `control_token` or explicit legacy debug precondition。
- Actor revalidates high-risk control before destructive effect。
- Route prompt commands are minted by daemon with active owner lease, never `unleased`。
- `control_epoch` gates high-risk actions; benign progress does not stale destructive control。
- Common expected control refusals return tool-success `ok=false` payloads。
- Prompt-not-submitted route state is visible as attention。
- Path containment, binding fail-closed, file claims, and daemon token boundary remain unchanged。

## Migration Notes

- Existing Codex prompts/skills should stop teaching manual lease fields。
- Old clients using `precondition` continue to work for one version as debug compatibility。
- New recommended flow for high-risk action:

```text
agentcall_session(view=summary) -> read control.token -> session_send(control_token=...)
```

For benign control:

```text
session_send(name, action=continue)
```

No manual key needed.
