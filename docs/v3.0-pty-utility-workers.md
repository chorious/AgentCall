# AgentCall v3.0: PTY Utility Workers

v3.0 将 AgentCall 收敛为主控 GPT/Codex 使用的本地 worker coordination interface。当前 live worker runtime 只保留 Claude Code PTY；ACP 退出当前产品面，相关 daemon runtime/supervisor 与 Python ACP driver 已删除。

核心目标是降低 Codex 上下文压力，让 Codex 通过 compact board、session summary 和 report 管理 Claude Code worker，而不是默认读取大段 raw terminal。

## 产品定位

- AgentCall 是 Codex 的本地控制面，不是 Claude Code 的替代品。
- Claude Code PTY 是当前唯一 live worker runtime。
- Codex 决定是否 handoff、是否请求 plan、何时验收 report。
- AgentCall 负责启动 PTY worker、提交 prompt、绑定 hooks、压缩状态、记录 reports 和暴露 board。
- GPT-Codex worker adapter 留给后续版本验证。

## 默认路线

`agentcall_route(mode=start,runtime=auto|pty)` 默认启动 utility PTY worker：

```text
claude --permission-mode auto
```

route result 会包含：

```json
{
  "runtime": "pty",
  "worker_kind": "utility",
  "pty_workflow": "normal",
  "workflow_status": "running",
  "phase": "execute",
  "permission_mode": "auto",
  "prompt_gate": {
    "status": "started_and_prompt_submitted | started_pending_prompt_ack | prompt_submit_failed"
  },
  "patience_policy": {
    "suggested_wait_seconds": 45,
    "do_not_retry_before_seconds": 60,
    "stall_threshold_seconds": 180
  }
}
```

`runtime=acp` 在 v3.0 是非法 runtime，不启动 child。

## Plan Mode

Plan mode 不是默认路线。Codex 认为任务需要先审查、澄清或制定计划时，可以显式传入：

```json
{
  "pty_workflow": "plan_then_auto",
  "initial_permission_mode": "plan"
}
```

Plan workflow 的控制语义：

- `ExitPlanMode` hook -> `plan_ready`
- `agentcall_session_send(action=approve_plan)`
- `agentcall_session_send(action=start_auto)`
- `agentcall_session_send(action=revise_plan)`

默认不用 plan mode 的原因是：v3.0 更重视低摩擦 worker 调度。复杂或高风险任务由 Codex 主动要求 plan，而不是让所有 worker 先进入计划等待。

## Hooks 的约束

v3.0 继续保留 hooks，并让 hooks 只提供高价值信号：

- `AGENTCALL_WRAPPER_SESSION` 绑定 PTY session 与 Claude hook session。
- `UserPromptSubmit` 用于确认 prompt 已真正提交。
- permission notification 会进入 board attention。
- `ExitPlanMode` 标记 plan gate 状态。
- 带 `allowed_paths` 的 PTY route 会在 `PreToolUse` 阶段做 path-scoped deny。

如果 `allowed_paths` 缺失，AgentCall 不默认阻断工作，但 summary 会标记 `containment=prompt_only`。

## Prompt Submit Gate

daemon 启动 PTY 后必须：

1. 等待 PTY/TUI ready。
2. 写入 prompt。
3. 等待 `UserPromptSubmit` hook ack。
4. 未 ack 时重试。
5. 返回明确状态：
   - `started_and_prompt_submitted`
   - `started_pending_prompt_ack`
   - `prompt_submit_failed`

这可以减少“session 创建了，但任务其实没有送进去”的误判。

## MCP 收敛

当前推荐工具：

- `agentcall_daemon`
- `agentcall_board`
- `agentcall_route`
- `agentcall_session`
- `agentcall_session_send`
- `agentcall_report`

`agentcall_route.runtime` 只接受：

```json
["auto", "pty"]
```

默认 schema 移除 ACP-only 字段：

- `adapter_command`
- `timeout_seconds`
- `template`
- `target_files`
- `max_reads`
- `max_writes`

`agentcall_board.section` 不再提供 `acp`。

## 状态观测

`agentcall_board(view=compact)` 默认返回：

- workspace
- live daemon PTY sessions
- attention
- recent route summaries
- recent report summaries

默认不返回 raw events、transcripts、claims 全量。

`agentcall_session` 默认返回 compact `llm_summary`：

```json
{
  "session": "...",
  "status": "...",
  "liveness_status": "...",
  "attention_status": "...",
  "workflow_status": "...",
  "mode": "...",
  "headline": "...",
  "needs_attention": false,
  "last_hook_status": "...",
  "claimed_files": [],
  "reports": [],
  "confidence": 0.9,
  "containment": "enforced | prompt_only"
}
```

clean tail 只在 `include=["clean_tail"]` 时返回。

## Patience Contract

Claude Code PTY worker 是异步后台 worker，不是同步函数调用。v3.0 在 route 和 session summary 中显式返回耐心契约：

- `suggested_wait_seconds`：启动或最近进展后建议等待多久再检查。
- `do_not_retry_before_seconds`：在该窗口内不要重复 route、重启或连续发送 `continue`。
- `last_progress_age_seconds`：距离 daemon 观察到 PTY 输出或状态更新的秒数。
- `stall_threshold_seconds`：超过该阈值后再考虑读取 clean tail 或请求简短状态。
- `patience_hint`：给 Codex 的自然语言提醒。

只有 `attention_status` 出现 `needs_permission`、`waiting_input`、`unbound`、`failed`，或 prompt gate 明确失败时，Codex 才应该立即介入。

## ACP 删除范围

v3.0 已删除当前实现里的 ACP live route：

- Rust daemon ACP client。
- Rust ACP background supervisor。
- Python ACP stdio client。
- Python ACP workflow driver。
- ACP parity test。

历史文档仍保留在 docs 中，方便追溯为什么放弃这条路线，但它们不是当前产品能力。

## 验收标准

- Codex 不需要理解 ACP。
- 默认 route 启动可见 PTY worker。
- 默认 PTY 是 auto mode。
- path containment 在带 `allowed_paths` 的 PTY route 下生效。
- prompt submit 不 silent fail。
- board compact 足够小。
- session summary 默认足够读。
- `runtime=acp` 不出现在 MCP schema、README 当前能力、runtime health 默认字段中。
- 显式 plan workflow 仍可用。
