# AgentCall MCP / Daemon Control Plane

AgentCall MCP 现在是稳定 stdio bridge，而不是业务实现层。它的职责是让 Codex 或其他 MCP client 发现 AgentCall，并把调用转发给 Rust daemon。

## 设计目标

- MCP transport 尽量稳定，不因为 route/session/report 业务更新而频繁重启。
- daemon 是业务工具定义和执行的权威。
- daemon 不可用时，MCP 仍能暴露 `agentcall_daemon` 用于启动 daemon。
- 不保留 Python workflow 作为默认 MCP 路线。

## 组件分工

```text
Codex / MCP client
  -> agentcall-mcp stdio bridge
      -> agentcall_daemon 本地 bootstrap
      -> GET  /api/mcp/tools
      -> POST /api/mcp/call
          -> Rust daemon
              -> board / route / session / report
```

`agentcall-mcp` 本地固定工具只有：

```text
agentcall_daemon
```

daemon 动态提供 canonical 工具：

```text
agentcall_board
agentcall_route
agentcall_session
agentcall_session_send
agentcall_report
```

## 更新策略

正常业务更新只改 daemon：

1. `cargo build -p agentcall-daemon`
2. 重启 daemon。
3. MCP transport 不需要重启。

只有 MCP stdio 协议外壳本身变化时，才需要：

1. `cargo build -p agentcall-mcp`
2. 重启 Codex MCP transport 或开启新 Codex 会话。

已经 `Transport closed` 的旧 MCP 连接无法被代码热修复；需要一次 MCP 重连。v0.8.1 的意义是把后续高频业务变化从 MCP exe 中剥离出去。

## 构建

```powershell
cargo build -p agentcall-daemon -p agentcall-mcp
```

## 启动

daemon：

```powershell
target\debug\agentcall-daemon.exe --workspace E:\Project\AgentCall --port 3293
```

MCP stdio：

```powershell
target\debug\agentcall-mcp.exe --workspace E:\Project\AgentCall --daemon-url http://127.0.0.1:3293
```

## Smoke Test

PowerShell:

```powershell
$requests = @(
'{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}',
'{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}',
'{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"agentcall_route","arguments":{"objective":"smoke recommend","mode":"recommend","runtime":"auto","estimated_minutes":5,"estimated_files":1}}}'
) -join "`n"

$requests | target\debug\agentcall-mcp.exe --workspace E:\Project\AgentCall --daemon-url http://127.0.0.1:3293
```

Expected evidence:

- `initialize` returns `serverInfo.name = agentcall-mcp`。
- `tools/list` returns `agentcall_daemon` plus daemon-provided canonical tools。
- `agentcall_route` returns a daemon-created route record。
