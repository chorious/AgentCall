# AgentCall v3.1 MCP

AgentCall v3 exposes the v2 workflow runtime through a Rust MCP server. v3.1
makes Claude ACP the default live workflow path exposed through MCP.

The goal is discoverability: another process or agent should not need to know
AgentCall's Python module layout. It can ask MCP for tools, call a workflow, and
inspect reports/state evidence.

## Server

Build:

```powershell
cargo build -p agentcall-mcp
```

Run over stdio:

```powershell
target\debug\agentcall-mcp.exe --workspace E:\Project\AgentCall
```

The server implements the MCP stdio JSON-RPC surface:

- `initialize`
- `tools/list`
- `tools/call`

## Tools

```text
agentcall_capabilities
agentcall_report_schema
agentcall_workflow_simulate
agentcall_workflow_inspect
```

`agentcall_workflow_simulate` bridges into the existing v2 runtime:

```text
Rust MCP server
  -> python -m agentcall --root <root> workflow simulate --driver acp
  -> .agentcall task/report/state artifacts
```

The default driver is `acp`, which launches
`@agentclientprotocol/claude-agent-acp` over stdio JSON-RPC. It is not the
legacy `claude -p` path.

Free smoke tests can still request the deterministic fixture explicitly:

```json
{"driver": "scripted"}
```

## Codex App Registration

The local Codex config now contains:

```toml
[mcp_servers.agentcall]
command = 'E:\Project\AgentCall\target\debug\agentcall-mcp.exe'
args = ['--workspace', 'E:\Project\AgentCall']
startup_timeout_sec = 30
```

Codex loads MCP servers at session startup, so a running conversation may not
see the new tools until Codex starts a fresh session or reloads MCP config.

## Smoke Test

PowerShell smoke:

```powershell
$requests = @(
'{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"smoke","version":"0"}}}',
'{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}',
'{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"agentcall_workflow_simulate","arguments":{"root":".pytest_tmp_mcp_smoke","driver":"acp"}}}',
'{"jsonrpc":"2.0","id":4,"method":"tools/call","params":{"name":"agentcall_workflow_inspect","arguments":{"root":".pytest_tmp_mcp_smoke","task_id":"task-0001"}}}'
) -join "`n"

$requests | .\target\debug\agentcall-mcp.exe --workspace E:\Project\AgentCall
```

Expected evidence:

- `initialize` returns `serverInfo.name = agentcall-mcp`.
- `tools/list` returns the four AgentCall tools.
- simulate returns `status: accepted` and `reports: 2`.
- inspect returns planner/executor state timeline.

For a token-free smoke run, change the simulate arguments to
`{"root": ".pytest_tmp_mcp_smoke", "driver": "scripted"}`.
