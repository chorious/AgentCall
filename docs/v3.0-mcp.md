# AgentCall MCP / Daemon Control Plane

AgentCall MCP 是一个稳定 stdio bridge。它的职责是让 Codex 或其他 MCP client 接入 AgentCall，并把实际业务调用转交给 Rust daemon。

## 设计目标

- MCP transport 尽量稳定，不因 route/session/report 业务更新频繁重启。
- daemon 是工具定义和工具执行的权威来源。
- daemon 不可用时，MCP 只暴露 `agentcall_daemon` 用于本地 bootstrap。
- Python workflow 不作为默认 MCP 路线。

## 组件分工

```text
Codex / MCP client
  -> agentcall-mcp stdio bridge
      -> agentcall_daemon local bootstrap
      -> GET  /api/mcp/tools
      -> POST /api/mcp/call
          -> Rust daemon
              -> board / route / session / report
```

`agentcall-mcp` 本地固定工具只有：

```text
agentcall_daemon
```

daemon 动态提供当前 canonical 工具：

```text
agentcall_board
agentcall_route
agentcall_session
agentcall_session_send
agentcall_report
```

v3.0 中 `agentcall_route` 只支持 `runtime=auto|pty`。`runtime=acp` 是非法 runtime。

## 更新策略

正常业务更新只需要重启 daemon：

1. `cargo build -p agentcall-daemon`
2. 重启 daemon。
3. MCP transport 不需要重启。

只有 MCP stdio 外壳本身变化时才需要：

1. `cargo build -p agentcall-mcp`
2. 重启 Codex MCP transport 或打开新的 Codex 会话。

`Transport closed` 通常表示 Codex 当前 MCP stdio transport 没有重新绑定到新进程。v3.0 的方向是尽量把工具面变化放到 daemon 后面，减少这种重启成本。

## 本机 Daemon 配置

daemon 默认读取：

```text
config/agentcall.local.json
```

该文件不提交到 git。可提交的模板是：

```text
config/agentcall.example.json
```

Claude PTY 的 cwd 必须来自 local config 的 `claude_workspace`。如果缺失，daemon health 会返回配置缺失，route/session 启动应失败并提示补配置。

MCP 不需要、也不应该通过工具参数传本机 cwd。route 请求里的 `workspace` 是任务目标上下文，不是 Claude Code 启动目录。

## 构建

```powershell
cargo build -p agentcall-daemon -p agentcall-mcp
```

## 启动

daemon：

```powershell
target\debug\agentcall-daemon.exe --workspace E:\Project\AgentCall --port 3293
```

MCP stdio：

```powershell
target\debug\agentcall-mcp.exe --workspace E:\Project\AgentCall --daemon-url http://127.0.0.1:3293
```

## Smoke Test

PowerShell:

```powershell
$requests = @(
'{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}',
'{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}',
'{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"agentcall_route","arguments":{"objective":"smoke recommend","mode":"recommend","runtime":"auto"}}}'
) -join "`n"

$requests | target\debug\agentcall-mcp.exe --workspace E:\Project\AgentCall --daemon-url http://127.0.0.1:3293
```

Expected evidence:

- `initialize` returns `serverInfo.name = agentcall-mcp`。
- `tools/list` returns `agentcall_daemon` plus daemon-provided canonical tools。
- `agentcall_route` returns a daemon-created PTY route recommendation or route record。
