# AgentCall v2.2: ACP SOP Worker Gate

v2.2 把 ACP 从“自由 runtime”收敛为轻量化 SOP worker。Codex 不再负责判断任务“小不小”；daemon 只接受已经规约好的 SOP contract，并用权限策略把 ACP 关在可验证边界内。

## 设计结论

- ACP 适合“一次 prompt -> 一份 report -> 结束”的 SOP worker。
- PTY 适合复杂长任务、需要协商、需要可视化 handoff、或边界暂时说不清的任务。
- `runtime=auto` 不再使用 `estimated_minutes/files/loc` 打分选择 ACP。
- ACP 必须显式提供 SOP template、目标文件、报告路径、allowed paths 和验收标准。
- 无合法 SOP contract 时，route 返回 `needs_contract`，不会启动 ACP。

## 首版 SOP

v2.2 首版只支持 5 个模板：

- `read-and-report`：读取指定文件，写一份结构化报告。
- `evidence-check`：核对报告、日志、文件证据是否存在且一致。
- `contract-check`：对照 contract、acceptance criteria、实现文件给出 pass/fail/evidence。
- `diff-review`：审查指定 diff 或文件集合，输出 findings。
- `single-report-update`：更新一个既有 report 文件。

这些模板都最多只能写一个 `report_path`。`single-file-fix` 暂不进入 ACP 默认模板。

## Route Contract

`agentcall_route` 保持唯一入口，但新增可选字段：

```json
{
  "template": "read-and-report",
  "target_files": ["README.md"],
  "report_path": ".agentcall/reports/readme_check.md",
  "allowed_paths": ["."],
  "acceptance_criteria": ["report includes status, verdict, evidence"],
  "max_reads": 10,
  "max_writes": 1
}
```

`runtime=auto` 行为：

- 有合法 SOP contract：推荐或启动 ACP。
- 没有合法 SOP contract：返回 `needs_contract`，并建议补 contract 或显式使用 PTY。

## Daemon Enforcement

v2.2 不只把规则写进 prompt。daemon 在两层执行边界：

- ACP `session/request_permission` 使用 template-aware default-deny policy。
- Claude hooks 根据 `AGENTCALL_WRAPPER_SESSION` 找到 route contract，在 `PreToolUse` 拦截越界工具。

首版策略：

- `Write/Edit/MultiEdit/NotebookEdit` 只能命中 `report_path`。
- `Read/Grep/Glob` 如果暴露路径，必须落在 `target_files` 或 `allowed_paths` 内。
- `Bash` 只允许少量只读命令；写入、重定向、删除、移动、复制默认 deny。

## Report Contract

每个 SOP 的 prompt 都要求输出固定字段：

- `status`
- `summary`
- `verdict`
- `evidence`
- `files_read`
- `changed_files`
- `risks`
- `next_recommended_action`
- `context_sufficiency`

daemon 会验证 final text 或 `report_path` 内容。缺字段时 route 状态会落到 `failed_report_contract`，而不是把 raw text 当成已验收 report。

## Runtime Behavior

ACP route 在启动后快速返回 `started`，后台继续执行 daemon-owned ACP invocation。board/routes 中可以看到：

- `sop_status`
- `permission_denials`
- `report_contract_status`
- `checkpoint_due`
- `last_heartbeat_at`
- `last_progress_at`

v2.4 起，ACP 的实时后台状态写入 `.agentcall/state/acp_invocations.json`；route 只保留 compact projection，不再保存完整 `last_acp_update`。

PTY route 继续作为复杂任务主线，并在启动后自动提交 objective/context prompt，返回 `started_and_prompt_submitted` 或 `started_pending_prompt`。
