# AgentCall v5.4.4 Code Plan：Projection Truth + PTY Terminal Replay

Date: 2026-06-11

## Summary

v5.4.4 是 v5.4 主线完成后的 projection / board / PTY 观测热修版本。它不改 `docs/v5.4-code-plan.md`，也不重开 ACP/SDK/`claude -p`/background-agent 方向。

本计划原本以 `v5.4.1` 命名，但 GPTPro review 指出版本号会与早先拆分计划的语义冲突。为避免 Codex、release-check、CHANGELOG、后续报告混淆，本轮改名为：

```text
v5.4.4  Projection Truth + PTY Terminal Replay
```

本轮目标不是再修一批零散现象，而是建立三条硬不变量：

```text
Invariant 1: Terminal dominance
一旦 PTY/session 有 observed terminal event，后续 command bookkeeping 不能复活 liveness。

Invariant 2: Runtime/projection bucket separation
board 的 live/runtime 字段只能来自 daemon 内存 state.sessions；历史状态必须在 historical/projection bucket。

Invariant 3: PTY terminal replay, not transcript regex
PTY 可读上下文必须来自 terminal-emulator replay 后的 screen snapshot，而不是 raw transcript regex cleanup。
```

关键订正：

```text
raw PTY bytes
  -> raw replay ring / forensic artifact
  -> TerminalScreen VT replay
  -> screen_snapshot / screen_snapshot_tail
  -> optional light snapshot filter
  -> debug-readable terminal context

hooks/events/projection/routes/policy
  -> summary / board / attention / next_action
```

`screen_snapshot_tail` 只解决“看得清”。状态可信度继续由 hooks/events/projection/routes/policy 解决。

- `claude_workspace=D:\guKimi`;

- `prompt_gate.status=started_and_prompt_submitted`;
- `workspace_lease.mode=SharedReadonly`;
- 三个 worker 均输出 `AGENTCALL_SMOKE_*_OK`;
- stop 带 lease/precondition 后被接受；
- stop 后 daemon health 显示 owner/workspace lease 归零。

暴露问题：

| Check | Expected | Observed |
| --- | --- | --- |
| clean completion attention | none | `SubagentStop` 将干净 smoke 标成 `checkpoint_due` |
| stop后 summary | stopped/completed/terminal | 被晚到的 `command.awaiting_observation` 覆盖成 `working` |
| board live sessions | 仅真实 in-memory PTY sessions | 混入历史 projection、route-only projection、已停止 smoke |
| clean tail | LLM-friendly debug surface | raw transcript-derived cleaner 仍含 spinner glyph、decorative verbs、repaint 残片 |

关键结论：

```text
runtime truth 基本正确；
projection truth 与 board surface 有误导性；
raw transcript-derived clean_tail 抽象层错误，应被 PTY terminal replay 取代。
```

## Non-goals

- 不修改冻结的 `docs/v5.4-code-plan.md`。
- 不恢复 ACP/SDK 作为主线 runtime。
- 不引入 `claude -p` / Agent SDK / Claude background agents 作为本轮状态或 TUI 修复路线。
- 不把 xterm.js serialize、浏览器 viewer 或前端 framebuffer 当 daemon source of truth。
- 不用 `TERM=dumb` / `NO_COLOR` 作为主修复；它们最多是辅助降噪。
- 不让 raw terminal 或 clean_tail 成为默认状态来源。
- 不新增 MCP 工具。
- 不自动 kill / auto-hide 可见 PTY worker。
- 不把 Python 作为 live writer。
- 不用 string grep 代替 route/event/projection/policy 的结构化状态。
- 不把 lease release 单独当作进程 terminal proof。
- 不继续投入“raw transcript 正则清洗”作为主要方案。

## Invariant 1: Terminal Dominance

### Problem

三只 smoke worker stop 后，事件流已有 terminal evidence：

```text
pty.stop_requested
session.cleanup
pty.session_ended
owner_lease.released
workspace_lease.released
session.cleanup
```

但最后又出现：

```text
command.awaiting_observation
```

最终 summary 显示：

```text
liveness_status=working
turn_status=awaiting_observation
```

这说明 projection reducer 过度 last-event-oriented。

### Terminal Evidence Classes

Primary terminal evidence:

```text
pty.session_ended
process.exited / process_exited
session.actor_failed
session.writer_failed fatal
session.orphaned fatal
```

Secondary / corroborating evidence:

```text
session.cleanup
owner_lease.released
workspace_lease.released
```

Rules:

```text
If primary terminal evidence exists:
  liveness = stopped | killed | completed | failed

If only cleanup/lease release exists but no primary process/session exit:
  liveness = cleanup_observed
  attention = needs_runtime_reconcile
```

Rationale:

Lease release is resource state, not process truth. If lease release happens before process exit, AgentCall should surface a reconcile issue, not silently mark the session completed.

### Projection State

Add terminal metadata to projection:

```rust
projection.terminal: bool
projection.terminal_event_id: Option<String>
projection.terminal_global_seq: Option<u64>
projection.terminal_reason: Option<String>
projection.stop_intent: Option<StopIntent>
projection.last_command_status: Option<CommandStatusSummary>
```

Reducer rule:

```rust
if projection.terminal {
    update_command_metadata(event);
    update_last_seen(event);
    return projection; // do not change liveness_status or attention_status
}
```

`command.awaiting_observation` may update:

```text
last_command_status
last_command_id
```

It must not update:

```text
liveness_status
attention_status
next_recommended_action
turn_status away from terminal
```

After terminal, `turn_status` should be:

```text
terminal
```

not `awaiting_observation`.

### Stop Intent

`pty.stop_requested` / `pty.kill_requested` should write:

```json
{
  "stop_intent": {
    "initiator": "supervisor",
    "command_id": "...",
    "mode": "stop|kill",
    "requested_at_seq": 123
  }
}
```

Exit classification:

```text
if exit_code != 0 and stop_intent in [supervisor_stop, supervisor_kill]:
  liveness=stopped/killed
  attention=none
  exit_class=expected_nonzero_due_to_termination

if exit_code != 0 and no stop_intent:
  liveness=failed
  attention=failed

if exit_code == 0 and no stop_intent:
  liveness=completed
  attention=none
```

## Invariant 2: Runtime / Projection Bucket Separation

### Problem

`agentcall_daemon(status)` reported:

```text
active_pty_sessions=1
live_daemon_sessions=1
owner_leases.active=0
workspace_leases.active=0
```

But `agentcall_board(view=compact, section=sessions)` returned many entries under `live_daemon_sessions`, including:

- old completed workers;
- route-only projection IDs;
- stopped smoke workers;
- historical projection rows.

This makes Codex reason about ghost workers.

### Board Contract

Split board session buckets:

```json
{
  "runtime_sessions": [],
  "historical_sessions": [],
  "route_sessions": [],
  "attention": [],
  "legacy_detached_sessions": [],
  "counts": {
    "runtime": 0,
    "historical": 0,
    "routes": 0,
    "attention": 0,
    "legacy_detached": 0
  }
}
```

Definitions:

```text
runtime_sessions:
  only sessions in daemon memory state.sessions

historical_sessions:
  non-live session projections from store/projection index

route_sessions:
  route records and route-only projections, never pretending to be PTY sessions

attention:
  deduplicated actionable items from runtime/projection/route/policy

legacy_detached_sessions:
  explicit legacy/debug sessions only
```

Compatibility:

```json
{
  "live_daemon_sessions": "alias of runtime_sessions only",
  "live_daemon_sessions_deprecated_alias": true
}
```

MCP descriptions should say:

```text
live_daemon_sessions is a backward-compatible alias for runtime_sessions only.
Historical projection rows are in historical_sessions.
Route records are in route_sessions.
```

### Performance Rule

`board(view=compact)` must not scan raw event history.

Allowed sources:

```text
state.sessions
projection hot/index state
routes state
attention index
legacy detached index
```

Disallowed in compact board:

```text
raw event payloads
full transcript scan
full terminal replay
large hook JSON blobs
```

## Invariant 3: PTY Terminal Replay Replaces Regex Cleaning

### Problem

`clean_tail` still contains Claude Code TUI decoration and repaint residue:

```text
Puttering...
Blanching...
Ideating...
Brewed
Cooked
thinking
✻ ✶ ✢ ✽ ●
running stop hooks... 0/2
duplicated objective/rules blocks
dropped-letter repaint fragments
```

UTF-8 decode is healthy, but clean tail is still too noisy for LLM-friendly debug.

This is not primarily an encoding bug and not primarily a missing denylist. It is a layer bug:

```text
raw PTY transcript records every repaint frame;
Claude Code TUI rewrites the same screen with cursor movement and clear-line sequences;
regex sees all intermediate frames as text;
LLM receives fragments that never existed together on screen.
```

Therefore v5.4.4 must stop treating raw terminal transcript as the basis for the readable surface.

### Boundary Rule

`clean_tail` is deprecated debug/readability only.

It must never drive:

```text
summary.liveness_status
summary.attention_status
board.attention
policy state
route state
report_ready
```

State must come from:

```text
projection
events
routes
policy
leases
runtime state
report validators
```

### PTY Replay Pipeline

The daemon owns a per-session terminal emulator state:

```text
raw PTY bytes
  -> raw replay ring / artifact
  -> TerminalScreen VT replay
  -> current rendered screen buffer
  -> screen_snapshot
  -> screen_snapshot_tail
  -> optional light snapshot filter
```

The initial implementation should use `vt100::Parser` behind a small internal trait, so we can replace it later without changing session/summary contracts:

```rust
pub(crate) trait TerminalEmulator {
    fn process(&mut self, bytes: &[u8]);
    fn resize(&mut self, rows: u16, cols: u16);
    fn snapshot(&self, max_lines: usize) -> TerminalSnapshot;
}
```

First backend:

```rust
pub(crate) struct Vt100TerminalEmulator {
    parser: vt100::Parser,
    rows: u16,
    cols: u16,
    seq: u64,
}
```

`termwiz` remains a future replacement candidate if `vt100` is insufficient, but v5.4.4 must keep the first implementation small.

### Raw Replay Is Forensic

Raw bytes remain available for reproduction and debug:

```text
raw_output_tail        raw PTY bytes/text tail, debug only
screen_snapshot        rendered terminal screen state
screen_snapshot_tail   readable tail from rendered screen buffer
semantic_activity      projection/events/routes/policy activity, state source
```

Raw replay should keep chunk metadata so VT replay bugs are reproducible:

```rust
struct PtyRawChunk {
    seq: u64,
    ts_ms: u64,
    bytes_offset: u64,
    len: usize,
    rows: u16,
    cols: u16,
}
```

Resize events must be included in replay metadata:

```rust
struct PtyResizeEvent {
    seq: u64,
    ts_ms: u64,
    rows: u16,
    cols: u16,
}
```

### TerminalScreen State

Add:

```text
crates/agentcall-daemon/src/terminal_screen.rs
```

Core structure:

```rust
pub(crate) struct TerminalScreen {
    parser: vt100::Parser,
    rows: u16,
    cols: u16,
    seq: u64,
    last_snapshot_hash: u64,
    last_semantic_hash: u64,
    frames_seen: u64,
    volatile_frames: u64,
    meaningful_frames: u64,
    screen_unreliable: bool,
}
```

PTY reader must feed every chunk into both raw replay and `TerminalScreen`:

```text
1. append bytes to raw replay ring
2. broadcast raw bytes to viewer/websocket
3. process bytes through TerminalScreen
4. update bounded screen snapshot cache
```

Do not drop screen chunks silently. If an async screen actor is added later and falls behind, it must either apply backpressure, rebuild from raw replay, or mark `screen_unreliable=true`.

### Screen Snapshot Contract

New response shape:

```json
{
  "terminal": {
    "raw_output_tail_available": true,
    "screen_snapshot_available": true,
    "screen_snapshot": {
      "schema_version": 1,
      "seq": 12345,
      "rows": 40,
      "cols": 100,
      "mode": "alternate|normal",
      "cursor": {"row": 38, "col": 12, "hidden": false},
      "text": "... current rendered screen ...",
      "tail": [
        "Tool denied: Write backend/internal/domain.go",
        "1. Yes, allow",
        "2. No"
      ],
      "quality": {
        "source": "vt100",
        "volatile": true,
        "frames_seen": 98,
        "volatile_frames": 91,
        "meaningful_frames": 7,
        "lines_returned": 12,
        "lines_suppressed": 5,
        "screen_unreliable": false
      }
    }
  }
}
```

`mode=alternate` matters because Claude Code TUI can draw a dashboard in alternate screen. In alternate mode, the snapshot describes current screen, not transcript scrollback.

### Snapshot Filter Strategy

Filtering is allowed only after terminal replay has produced rendered screen lines.

Drop if:

```text
line has no path;
line has no alnum word length >= 3;
line has no semantic keyword;
glyph ratio > 0.4;
line is single-char / scattered repaint residue;
same normalized line repeats > N times within tail;
objective/rules block repeats with same hash.
```

Keep if line contains:

```text
AGENTCALL_SMOKE_
permission / allow / deny / denied
error / failed / panic / blocked
report / complete / done
path-like string
numbered menu option
tool name + status
```

The initial explicit decoration list may remain as a post-snapshot safety net:

```text
Puttering
Blanching
Ideating
Brewed
Cooked
thinking
✻
✶
✢
✽
●
⏵⏵ auto mode on
← for agents
```

But these rules must not operate directly on raw PTY transcript as the primary strategy.

### Frame Stability

Track semantic vs volatile frame changes:

```text
current_screen_snapshot:
  latest rendered terminal screen, may be spinner/volatile

last_semantic_screen:
  latest rendered screen whose normalized semantic hash changed
```

Default `agentcall_session(include=["summary"])` should not include large terminal text. Debug/tui views may include `screen_snapshot_tail`, preferably from `last_semantic_screen` when the current frame is volatile.

### Quality Metadata

Add quality metadata:

```json
{
  "source": "vt100",
  "raw_chunks_seen": 300,
  "frames_seen": 98,
  "volatile_frames": 91,
  "meaningful_frames": 7,
  "lines_returned": 12,
  "lines_suppressed": 5,
  "truncated": false,
  "confidence": "medium",
  "screen_unreliable": false,
  "semantic_markers_preserved": ["AGENTCALL_SMOKE_A_OK"]
}
```

The confidence is heuristic and must be presented as readability confidence, not state confidence.

## Projection Truth Add-on: Policy Denial Attention

### Problem

Earlier real logs showed repeated denied Write/Bash attempts while summary still looked safe:

```text
attention_status=none
needs_attention=false
next_recommended_action=inspect_session_debug
```

This is the same class of projection-truth bug: events say the worker is blocked, but summary says it is fine.

### Rule

Denied Write/Bash policy events must fold into:

```text
attention_status=blocked_by_policy
needs_attention=true
safe_to_wait=false
next_recommended_action=fix_path_policy_or_interrupt
```

This must work even when each denied target has `repeat_count=1`, as long as multiple distinct denied writes occur in a short window.

### Priority Ordering

Attention priority:

```text
failed/orphaned
needs_permission
blocked_by_policy
checkpoint_due
report_ready
none
```

Rules:

- benign `SubagentStop` cannot clear higher-priority attention;
- policy block cannot be cleared by spinner/TUI clean tail;
- permission notification remains above checkpoint;
- terminal failure remains above policy block.

### Tests

- `denied_write_sets_blocked_by_policy_attention`
- `repeated_distinct_denied_writes_fold_into_one_blocker`
- `subagent_stop_does_not_clear_existing_policy_block`
- `permission_notification_remains_above_checkpoint_due`

## SubagentStop Semantics

### Problem

Clean smoke workers completed their OK marker but were projected as:

```text
checkpoint_due
```

solely because of `SubagentStop`.

### Rule

`SubagentStop` is benign by default.

It may set `checkpoint_due` only when explicit:

```text
route.result.checkpoint_contract.subagent_stop_requires_review = true
payload.checkpoint_request = true
payload.agentcall_checkpoint = true
```

Do not infer checkpoint from weak signals like:

```text
role contains subagent
task_id exists
Claude internal agent behavior
TUI wording
```

For benign `SubagentStop`:

```text
hook_status=observed
liveness_status unchanged
attention_status unchanged
last_progress_brief="SubagentStop observed (benign)"
```

Do not introduce a new vague `working_done` status.

### Checkpoint Contract

Route result schema:

```json
{
  "checkpoint_contract": {
    "subagent_stop_requires_review": true,
    "source": "route.child_task",
    "expected_report_path": "..."
  }
}
```

### Tests

- `subagent_stop_without_checkpoint_contract_is_benign`
- `subagent_stop_with_route_checkpoint_contract_sets_checkpoint_due`
- `subagent_stop_with_payload_checkpoint_sets_checkpoint_due`
- `subagent_stop_does_not_change_liveness_without_terminal_event`
- `subagent_stop_does_not_clear_existing_policy_block`

## Acceptance Criteria

Projection:

- Once a session has observed `pty.session_ended` / `process_exited`, later `command.awaiting_observation` cannot change `liveness_status` away from terminal.
- `SubagentStop` without checkpoint contract does not set `attention_status=checkpoint_due` and does not clear higher-priority attention.
- Denied Write/Bash policy events fold into `blocked_by_policy`, including repeated distinct targets.
- Supervisor stop/kill with nonzero exit is classified by stop intent, not raw exit code alone.
- Natural nonzero exit without stop intent remains failed.

Board:

- `runtime_sessions` and `live_daemon_sessions` contain only in-memory `state.sessions`.
- Historical/projection sessions never appear in `live_daemon_sessions`.
- Route-only records never appear in runtime session buckets.
- Board runtime count equals daemon health `active_pty_sessions` after cleanup.

PTY Terminal Replay:

- Summary and TUI status do not depend on `clean_tail`, raw output, or screen snapshot text.
- daemon maintains per-session `TerminalScreen` by replaying PTY bytes through a VT parser.
- `screen_snapshot_tail` is generated from rendered terminal framebuffer, not raw transcript regex cleanup.
- Intermediate cursor repaint frames do not appear in `screen_snapshot_tail`.
- Permission menus, smoke OK markers, reports, errors, and denied-policy lines remain visible in `screen_snapshot_tail`.
- Raw PTY bytes remain available for forensic debug.
- Old `clean_tail` is marked deprecated/debug-only and points to `terminal.screen_snapshot_tail`.
- Terminal snapshot quality reports source, mode, frames, volatility, truncation, and `screen_unreliable`.

Smoke:

- Three-worker read-only smoke passes.
- One natural-completion worker passes.
- One policy-denial fixture/session projects `blocked_by_policy`.
- Failed smoke runs save event/projection/board/terminal snapshot artifacts for diagnosis.
- `python agentcall.py release-check` passes.
