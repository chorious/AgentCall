# AgentCall v5.4 Final Plan: Session Observation, Lifecycle, Policy, Local Boundary

## Plan Freeze

This document is the final v5.4 plan. Until all acceptance criteria in this file are completed, agents must not modify this plan or split it into new v5.4.x plans.

If new evidence appears during implementation:

- record it in `docs/reports/`;
- fix code or tests when it fits this plan;
- report a blocker when it conflicts with this plan;
- do not rewrite this plan unless the user explicitly lifts the freeze.

## Summary

v5.4 merges the previous v5.4, v5.4.1, v5.4.2, and v5.4.3 drafts into one mainline. The goal is to make AgentCall usable under real Codex supervision pressure by fixing the four connected problems that surfaced in live logs:

- `agentcall_session` returns too much mixed data, so Codex pays a high context and latency cost.
- route start and session control can block or mislead Codex because lifecycle effects are not separated from command dispatch.
- policy denials, path roots, binding trust, and stale leases are not projected clearly enough.
- local daemon and MCP boundaries still allow oversized input, raw payload leakage, and legacy ambiguity.

The core product rule for v5.4:

```text
TUI and Codex must read compact projection/state, not raw terminal or raw hook payload.
```

## Non-Goals

- Do not revive ACP/SDK as a runtime direction.
- Do not make raw terminal output a status source.
- Do not make `clean_tail` determine liveness, attention, blockers, permissions, report readiness, or next action.
- Do not add new MCP tools when `board / route / session / session_send / report` can carry the behavior.
- Do not add Python live writers for events, claims, routes, sessions, projections, bindings, or leases.
- Do not declare hook policy an OS sandbox.
- Do not implement remote multi-user authentication. v5.4 only hardens local loopback daemon use.
- Do not keep SDK/runtime fields public if the implementation is not real.
- Do not treat legacy Python compatibility checks as release readiness.

## Phase A: Compact Observation Contract

### A1. Hard Session Views

`agentcall_session` must support hard `view` contracts:

```text
summary | tui | events | debug | raw
```

For `view=summary`, `view=tui`, and `view=events`, the response schema must be stable and must not depend on legacy `include`.

Legacy `include` may be translated internally, but it must not change the default raw policy:

- `view=summary`: ignore `include` except for debug references; never inline raw terminal, raw hook payload, full prompt, tool stdout, or Write/Edit content.
- `view=tui`: ignore `clean_tail` and raw includes; return dashboard data only.
- `view=events`: return compact events by default; raw payload only with `detail=raw`.
- `view=debug`: may return clean tail and debug references, still budgeted.
- `view=raw`: explicit large/debug surface, paginated and budget-tagged.

Code targets:

- `crates/agentcall-daemon/src/mcp.rs`
- `crates/agentcall-daemon/src/summary.rs`
- `crates/agentcall-mcp/src/tools.rs`

### A2. Response Budgets

Every session view response must include a `budget` object.

Hard budgets:

```text
summary target <= 4 KB, hard cap <= 8 KB
tui target <= 12 KB, hard cap <= 20 KB
events compact target <= 24 KB, hard cap <= 48 KB
debug/raw may exceed, but must be paginated and budget-tagged
```

Budget shape:

```json
{
  "budget": {
    "view": "tui",
    "max_bytes": 20480,
    "estimated_bytes": 7342,
    "truncated": false,
    "omitted": {
      "raw_events": 18,
      "tool_outputs": 3,
      "tool_inputs": 5,
      "terminal_tail": true
    }
  }
}
```

`view=tui` is a dashboard projection. It must be generated from:

1. session projection;
2. route record;
3. runtime binding;
4. policy denials;
5. compact recent events;
6. live session info;
7. terminal clean tail only as debug reference.

Top-level schema:

```json
{
  "schema_version": 1,
  "view": "tui",
  "session": "rvk-par-vic2-import",
  "header": {},
  "status": {},
  "paths": {},
  "activity": [],
  "current_blocker": null,
  "next_action": {},
  "counters": {},
  "cursors": {},
  "budget": {},
  "debug_refs": {}
}
```

`header` must include:

```json
{
  "title": "rvk-par-vic2-import",
  "runtime": "pty",

    "last_event_seconds": 2,

```

`status` must include:

### A5. Compact Events

Compact events are the default for `view=events`.

Required fields:

```json
{
  "event_id": "evt-038793",
  "global_seq": 38793,
  "session_seq": 18,
  "event_type": "hook.PreToolUse",
  "kind": "policy_denial",
  "severity": "warning",
  "actionability": "requires_supervisor",
  "summary": "Write denied: backend/internal/vic2/domain.go",
  "raw_ref": "evt-038793"
}
```

Enums:

```text
severity: trace | info | warning | error | critical
actionability: none | observe | wait | requires_supervisor | terminal
```

Default compact events must omit:

- full user prompt;
- tool stdout/stderr;
- Write/Edit/MultiEdit content;
- raw hook payload;
- large objective text.

Examples:

- `pty.session_started`: keep command, `process_cwd`, `target_workspace`.
- `hook.UserPromptSubmit`: keep prompt char count and `raw_ref`, not prompt text.
- allowed `Bash`: keep command summary, not full output.
- `hook.PostToolUse`: keep stdout/stderr line counts and artifact refs.
- denied `Bash`: keep denial reason and short command summary.
- denied `Write`: keep path, reason, diagnosis, not file content.

Code target:

- add or centralize `compact_event(event, context) -> CompactEvent`.

### A6. Activity And Blocker Folding

Activity building rules:

```text
input: latest 50 compact events
output: max 10 activity items
```

Rules:

- include latest blocker;
- include latest progress 1 to 3 items;
- group repeated denials;
- hide command accepted/completed by default;
- keep only latest prompt submitted;
- keep only PostToolBatch summaries;
- pin report ready when present.

Repeated denial rule:

```text
If there are >= 2 denied Write/Bash events in the latest 60 seconds for the same wrapper_session:
  attention = blocked_by_policy
  current_blocker.kind = policy
  safe_to_wait = false
```

This must apply even when every individual target has `repeat_count=1`.

Current blocker priority:

1. actor/writer/reader failed;
2. unexpected process exit;
3. `needs_permission`;
4. active policy denial;
5. latest denied Write/Bash repeated in a short window;
6. `prompt_ack_missing`;
7. `report_ready`;
8. no blocker.

### A7. Terminal Cleaner Boundary

Terminal output is split into:

```text
raw_replay_tail: raw PTY debug only
screen_snapshot: optional VT parser output
semantic_tail: compact-events-derived activity, not terminal-derived
```

`clean_terminal_text` remains useful for debug preview only.

It must not drive:

- liveness;
- attention;
- blocker;
- permission;
- report readiness;
- next action.

Terminal decoration filter should drop:

- spinner-only lines;
- one or two character glyph lines;
- low alphanumeric-ratio repaint fragments;
- repeated repaint fragments;
- scattered layout artifacts.

It should keep lines containing:

- permission or menu keywords;
- denied, failed, error, report, complete;
- path-like strings;
- meaningful command prompt or command output.

Code targets:

- `crates/agentcall-daemon/src/terminal.rs`
- `crates/agentcall-daemon/src/summary.rs`

## Phase B: Short Transactions And Lifecycle

### B1. Route Start Is A Short Transaction

`agentcall_route(mode=start, runtime=pty)` must not block on `UserPromptSubmit`.

Route start must:

- validate all context before spawning;
- dispatch prompt exactly once per idempotency key;
- never send an automatic second prompt inside the same MCP call;
- return a route id, session name, and prompt command id quickly;
- let projection/board/session observe hook ack later.

If no hook ack is observed during the short start transaction, return:

```json
{
  "status": "started_pending_prompt_ack",
  "prompt": {
    "command_id": "cmd-...",
    "idempotency_key": "route-prompt-rvk-par-vic2-import-1",
    "dispatched": true,
    "ack_expected": "hook.UserPromptSubmit",
    "ack_deadline_hint_seconds": 15
  },
  "next_observation": "agentcall_session(view=summary)"
}
```

TUI must show:

- `route_status=started_pending_prompt_ack` before `UserPromptSubmit`;
- `route_status=prompt_submitted` after hook ack;
- `route_status=prompt_ack_missing` after ack deadline, not `worker_failed`.

Code targets:

- `crates/agentcall-daemon/src/routes.rs`
- `crates/agentcall-daemon/src/projection.rs`
- `crates/agentcall-daemon/src/summary.rs`

### B2. Command Status Stages

Do not return a vague `accepted` for lifecycle commands.

Use:

```text
received | validated | queued | dispatched | effect_observed | failed
```

Stop response example:

```json
{
  "ok": true,
  "command_status": "dispatched",
  "effect": "stop_requested",
  "lease_release": "pending_process_exit",
  "next_observation": "process_exited"
}
```

Kill response example:

```json
{
  "ok": true,
  "command_status": "dispatched",
  "effect": "kill_requested",
  "process_tree_signal": "sent",
  "lease_release": "pending_process_exit"
}
```

### B3. Actor Priority

Actor command priority:

```text
Priority 0: KillSession
Priority 1: StopSession, InterruptTurn
Priority 2: Resize, SelectOption
Priority 3: SendInput, RequestReport, Continue
Priority 4: RawWrite / terminal aux response
```

Raw terminal auxiliary output must not block control-path commands. Prefer separate writer/aux channel or strict queue budget.

### B4. Stop And Kill Semantics

Stop and kill are distinct:

- stop: graceful request, lease remains active until process exit is observed;
- kill: forceful process-tree termination, lease remains active until waiter observes exit.

Session map removal and owner quota release happen only on observed process exit.

Actor panic or writer failure must project:

- `session.actor_failed`;
- `session.writer_failed`;
- `attention=terminal`;
- `safe_to_wait=false`.

Code targets:

- `crates/agentcall-daemon/src/actor.rs`
- `crates/agentcall-daemon/src/commands.rs`
- `crates/agentcall-daemon/src/session.rs`
- `crates/agentcall-daemon/src/ownership.rs`

## Phase C: Policy, Binding, And Path Hardening

### C1. Destructive Preconditions

Destructive actions require full precondition:

- `projection_last_session_seq`;
- `owner_lease_id`;
- `lease_generation`;
- `owner_id`.

Applies to:

- stop;
- kill;
- interrupt;
- approve plan;
- start auto;
- any future destructive or irreversible session command.

Validation rules:

```text
provided projection_last_session_seq == current projection_last_session_seq
provided owner_lease_id == active owner lease id
provided lease_generation == active owner lease generation
lease status is active
lease is not expired
owner_id matches lease owner_id
```

Error codes:

```text
rejected_missing_precondition
rejected_stale_projection
rejected_stale_lease
rejected_expired_lease
rejected_owner_mismatch
```

Empty `{}` is rejected.

### C2. Lease Identity

Lease id format:

```text
lease-{session_id}-{generation}-{nonce8}
```

Example:

```text
lease-rvk-par-vic2-import-2-a91f3c2b
```

Rules:

- `lease_generation` increments on reacquire/recover;
- released and expired leases cannot validate destructive commands;
- daemon restart must not make stale generation look current.

### C3. Binding Trust

Route worker policy must fail closed when binding is not trusted.

Rules:

```text
if route record exists but wrapper binding is missing/untrusted:
  Write/Edit/MultiEdit/Bash-write => deny with binding_untrusted

if no route record and session was created through direct debug API:
  default read-only
  write requires explicit debug_allow_write=true and local token
```

Summary/TUI must expose:

- `binding.trusted`;
- `binding.source`;
- `binding.last_hook`;
- missing events;
- last hook timestamp.

### C4. Root Coordinates

Every route must materialize roots at route creation.

Definitions:

```text
process_cwd / claude_workspace: where Claude Code runs
target_workspace: task/project root
scratch_root: writable scratch root for this wrapper session
```

Containment record:

```json
{
  "roots": {
    "process_cwd": "D:\\guKimi",
    "target_workspace": "E:\\GameProject\\RKV\\.agentcall-parallel\\vic2",
    "scratch_root": "D:\\guKimi\\.agentcall\\workspaces\\rvk-par-vic2-import"
  },
  "writable_roots": [
    {
      "kind": "scratch",
      "display": ".agentcall/workspaces/rvk-par-vic2-import",
      "abs": "D:\\guKimi\\.agentcall\\workspaces\\rvk-par-vic2-import"
    },
    {
      "kind": "report",
      "display": "report.md",
      "abs": "E:\\GameProject\\RKV\\.agentcall-parallel\\vic2\\report.md"
    }
  ]
}
```

Do not resolve policy from ambiguous hook-time relative paths like `.`, `report.md`, or `.agentcall/workspaces/...` without route materialization.

### C5. Path Diagnosis

Add:

```rust
diagnose_path(target, roots, policy) -> PathDiagnosis
```

Windows lexical normalization:

- replace `\` with `/`;
- lowercase drive letter and path for comparison;
- remove trailing slash;
- normalize `.` and `..` lexically;
- if path is relative, resolve against both `process_cwd` and `target_workspace`;
- do not require path existence.

Root class:

```text
scratch_root
report_path
target_workspace
process_cwd
outside_known_roots
```

Write/Edit rule:

```text
allowed if target is inside any writable_root.abs
denied otherwise
```

Bash rule:

```text
allowed only when the command is read-only under the bash policy;
non-read-only Bash is denied even if cwd is inside an allowed root.
```

Path diagnosis shape:

```json
{
  "target": {
    "raw": "D:/guKimi/.agentcall/workspaces/rvk-par-vic2-import/backend/internal/vic2/domain.go",
    "normalized": "d:/gukimi/.agentcall/workspaces/rvk-par-vic2-import/backend/internal/vic2/domain.go",
    "root_class": "scratch_root",
    "relative_to_root": "backend/internal/vic2/domain.go"
  },
  "roots": {
    "process_cwd": "D:/guKimi",
    "target_workspace": "E:/GameProject/RKV/.agentcall-parallel/vic2",
    "scratch_root": "D:/guKimi/.agentcall/workspaces/rvk-par-vic2-import"
  },
  "matched_writable_root": "scratch_root",
  "decision_expected": "allow",
  "decision_actual": "deny",
  "diagnosis": "target is inside scratch_root; denial likely caused by comparing against target_workspace-relative path"
}
```

Code targets:

- `crates/agentcall-daemon/src/hooks.rs`
- `crates/agentcall-daemon/src/routes.rs`

## Phase D: Local Boundary And Legacy Cleanup

### D1. MCP Input Cap

MCP stdio bridge must reject oversized input before parsing huge payloads.

Constant:

```text
MAX_MCP_INPUT_LINE_BYTES = 1 MiB
```

Behavior:

- line larger than limit returns controlled JSON-RPC error (`-32600` or explicit custom code);
- transport remains alive;
- subsequent valid ping/request still works.

Code target:

- `crates/agentcall-mcp/src/main.rs`

### D2. Unified Redaction

Add a shared redaction path:

```rust
redact_for_surface(value, Surface::Summary | Surface::Tui | Surface::Events | Surface::Debug | Surface::Raw)
```

Rules:

- Summary/TUI never inline raw prompt, tool stdout/stderr, or Write/Edit content.
- Events compact stores `raw_ref`, counts, and short summaries.
- Debug may include clean tail and artifact refs.
- Raw requires explicit view/detail and pagination.
- `mcp.tool_called` must not log full `objective`, `text`, prompt, or large args by default.

Code targets:

- `crates/agentcall-daemon/src/mcp.rs`
- `crates/agentcall-daemon/src/hooks.rs`
- `crates/agentcall-daemon/src/state.rs`
- `crates/agentcall-daemon/src/summary.rs`

### D3. Local Token And Origin Boundary

Default mode:

```text
dev_open_loopback=false
```

Rules:

- token required for all POST, WebSocket, and mutating APIs;
- `GET /board` static page may load, but JavaScript API calls require token;
- MCP bridge reads token from config or env;
- Origin/Host checks are supplementary, not the sole security boundary.

Code target:

- `crates/agentcall-daemon/src/http.rs`

### D4. Path Traversal

All path-taking endpoints must canonicalize or lexically normalize paths before access.

Rules:

- context/transcript/report indexing cannot escape workspace or configured allowed directories;
- `..` traversal is denied when outside allowed roots;
- raw artifact lookup uses ids or refs, not arbitrary path strings.

### D5. Legacy And Dead Field Cleanup

Release check must separate:

```text
runtime-critical Rust checks
plugin schema checks
legacy Python compatibility checks
```

Legacy checks passing must not print or imply:

```text
runtime protected
release ready
```

If SDK runtime is not implemented:

- remove SDK runtime from public MCP schema;
- keep only internal hidden flags if needed.

If `estimated_minutes`, `estimated_files`, `estimated_loc`, or `risk` are ignored:

- mark them `deprecated_ignored` in schema descriptions; or
- remove them from start path public schema.

Code targets:

- `scripts/`
- `agentcall.py`
- `crates/agentcall-mcp/src/tools.rs`
- `crates/agentcall-daemon/src/routes.rs`
- `README.md`
- `CHANGELOG.md`

## Test Plan

### Session Contract

- `agentcall_session(view=summary)` does not include `clean_tail`, raw terminal, raw events, tool response, prompt text, or Write content.
- `agentcall_session(view=tui)` returns bounded dashboard data under hard cap.
- `agentcall_session(view=events)` returns compact events by default.
- `detail=raw` is required for raw event payload.
- Legacy `include` cannot force raw into `summary/tui/events`.

### Projection And TUI

- `session_summary_folds_denied_writes_to_blocked_by_policy`.
- `compact_events_omit_prompt_and_write_content`.
- `tui_view_does_not_include_clean_tail_or_raw`.
- `path_diagnosis_identifies_scratch_root_target`.
- `repeated_distinct_denials_group_into_one_blocker`.
- `terminal_glyph_filter_drops_spinner_repaint`.

### Lifecycle

- route start returns before hook ack wait budget.
- route prompt dispatches exactly once per idempotency key.
- prompt ack later updates route/session projection.
- prompt ack missing becomes `prompt_ack_missing`, not worker failed.
- stop does not release lease until process exit.
- kill terminates process tree and releases lease only after waiter confirms exit.
- actor panic writes failed projection.
- high-priority stop/kill/interrupt bypass normal send queue.

### Preconditions And Policy

- destructive command with missing precondition is rejected.
- stale projection seq is rejected.
- stale lease generation is rejected.
- expired/released lease is rejected.
- owner mismatch is rejected.
- untrusted route worker binding denies write/edit/bash-write.
- scratch-root target is diagnosed correctly.
- path traversal outside allowed roots is denied.

### Local Boundary

- MCP bridge rejects input above 1 MiB and stays alive for the next request.
- mutating HTTP/WS without token is rejected.
- Origin/Host check rejects obvious browser misuse.
- `mcp.tool_called` is redacted and budgeted.
- raw logs/artifacts are bounded or rotated.
- release-check separates runtime-critical Rust checks from legacy Python compatibility.

## Acceptance Criteria

- Codex default supervision uses `agentcall_session(view=summary)` without large payloads.
- TUI renders `view=tui` without parsing raw hook JSON or raw terminal output.
- Raw payload is available only through explicit debug/raw view with pagination and budget.
- Policy denial loops become `attention=blocked_by_policy` with `safe_to_wait=false`.
- Route start cannot block MCP on hook ack or dispatch duplicate prompts.
- Stop, interrupt, and kill have priority and distinct lifecycle semantics.
- Stale Codex turns cannot perform destructive commands without fresh projection and active lease.
- Path denial output explains process cwd, target workspace, scratch root, matched root, and expected vs actual decision.
- Local MCP/HTTP daemon paths have input caps, redaction, token boundary, and traversal protection.
- Old v5.4.1/v5.4.2/v5.4.3 split-plan drafts are no longer treated as active plans.
