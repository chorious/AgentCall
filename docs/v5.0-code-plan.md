# AgentCall v5.0 Code Plan：Projection Fast Path + Codex 协同契约

## Summary

v5.0 只解决一件事：**Codex 默认看状态必须快、准、小，并且不触碰 cold path**。

GPT Pro 三轮反馈的共同结论是：v5.0 可以先做 projection fast path，不必直接完整 actor/event-store/SQLite 重构；但 projection 不能是无纪律缓存补丁。v5.0 必须同时定义最小事件外壳、projection reducer contract、projection stale 规则、idempotency 规则、MCP 响应边界和 tool safety 分级。

一句话：

```text
v5.0 让 Codex 不再靠猜；但不重写 runtime。
```

## Non-goals

- 不引入 Redis/Kafka/Postgres。
- 不引入 SQLite 作为默认 store。
- 不重命名现有 MCP tools。
- 不实现完整 SessionActor。
- 不让 Python 成为 live state writer。
- 不恢复 ACP 作为默认 runtime。
- 不让 TUI parser 改 authoritative liveness/attention。
- 不在 compact/attention path 里读取 transcript、plan file、raw PTY log、legacy sessions、full reports。
- 不把 `call_api` 超级工具作为主入口。
- 不允许 summary/mcp 查询路径写 projection。
- 不允许 write command 在缺少 `idempotency_key` 时继续执行。
- 不允许 MCP stdout 输出任何 timing/debug/log 内容。

当前主要慢路径：

- `summary.rs::attention_items()` 会对 live sessions 调用完整 `session_summary()`。

- `mcp.rs::session_has_seen_hook_event()` 和 summary 中同名逻辑都会读 events tail。
- `agentcall-mcp::daemon_client::daemon_request()` 无 connect/read/write timeout。
- `agentcall-mcp::protocol::tool_text()` 对返回值 `to_string_pretty`，没有 response cap。

## Hard Corrections From GPT Pro

本版订正后，v5.0 增加 9 条硬边界：

1. `EventEnvelopeV1` 必须拆 `global_seq` 与 `session_seq`，避免 daemon 全局 cursor 和 session-local precondition 混用。
2. Projection 只能通过 event reducer 更新；`summary.rs` / `mcp.rs` 只能读 projection。
3. write command 没有 `idempotency_key` 必须拒绝；destructive command 还必须带 precondition。
4. MCP timing/debug/log 只能写 file 或 stderr，严禁写 stdout。
5. MCP tool response 默认 compact JSON + cap；debug 才允许 pretty。
6. Projection reducer 是唯一更新入口；任何“现场补 projection”的 helper 都只能用于 bootstrap/debug，并且必须返回 `projection_stale=true`。
7. `session_send` 的 P0 idempotency 不允许“先 warning 后执行”；只要会写入 PTY 或改变 session 状态，就必须先通过 idempotency / precondition gate。
8. MCP bridge 只负责协议翻译和超时保护，不得在 bridge 层做 cold-path 聚合、文件扫描或大对象格式化。
9. MCP tool safety 要在 schema/description/skill/daemon 四层一致表达；`KillSession`、`StopSession`、`InterruptTurn` 不能只靠 Codex 自觉谨慎。

这些边界是开工条件，不是实现建议。若某条暂时做不到，计划必须显式降级验收，而不是在代码里保留隐式 fallback。

## Implementation Gate

v5.0 不以“看起来更快”为验收，而以默认 MCP 请求是否彻底脱离 cold path 为验收。任何 PR 合并前必须满足：

```text
1. board/session 默认路径只能读 projection；不得读 transcript、plan file、raw PTY log、full reports。
2. projection 只能由 event reducer 更新；summary/mcp/tools 不得现场修复 projection。
3. compact/attention response 必须 compact JSON + cap，不能 pretty dump 大对象。
4. write/side-effect command 缺 idempotency_key 必须拒绝，不能 warning 后继续执行。
5. stdout 只能承载 MCP JSON-RPC response；timing/debug/log 只能写文件或 stderr。
```

如果为了兼容旧状态需要 bootstrap projection，只允许在 daemon startup 或显式 debug 路径执行，并且结果必须带 `projection_stale=true`。v5.0 的目标不是“把旧信息再扫一遍扫快点”，而是让 Codex 默认只读已经投影好的小状态。

## Target Public Semantics

MCP 工具名保持不变：

```text
agentcall_daemon
agentcall_board
agentcall_route
agentcall_session
agentcall_session_send
agentcall_report
```

但语义冻结为：

- `agentcall_board`：默认 projection-only；`compact+attention` 必须 hot path。
- `agentcall_session`：默认 projection-only；`include` 才展开 clean tail / plan / events / artifacts / policy / metrics / debug。
- `agentcall_session_send`：提交 command；必须支持 `idempotency_key` 和 `precondition`，不承诺 Claude 已处理。
- `agentcall_route`：创建/绑定 worker session；返回 session/run/lease/projection，不等待任务完成。
- `agentcall_report`：读取 report projection / artifact，不默认 tail raw output。

Tool safety 语义必须与 MCP schema、daemon gate、Codex skill 保持一致：

```text
read-only:
  agentcall_daemon
  agentcall_board
  agentcall_session default
  agentcall_report read

side-effect:
  agentcall_route
  agentcall_session_send SendInput
  agentcall_session_send SelectOption
  agentcall_session_send RequestReport

destructive:
  agentcall_session_send InterruptTurn
  agentcall_session_send StopSession
  agentcall_session_send KillSession
```

`side-effect` 与 `destructive` 不只是文档标签；daemon 必须执行 idempotency / precondition / owner gate。Codex skill 只是让 GPT 少犯错，不能作为唯一保护层。

### Projection Reducer Contract

Projection 更新必须集中在 `projection.rs` 的 reducer：

```rust
pub(crate) struct ProjectionUpdate {
    pub projection: SessionProjectionV1,
    pub changed: bool,
    pub reason: String,
}

pub(crate) fn apply_event_to_projection(
    previous: Option<SessionProjectionV1>,
    event: &EventEnvelopeV1,
) -> ProjectionUpdate;
```

硬规则：

```text
events.rs / projection.rs 可以 update projection。
summary.rs / mcp.rs 只能 read projection。
board/session 默认路径不得调用 projection_from_existing_session()。
projection_from_existing_session() 只能用于 daemon startup bootstrap，且必须 projection_stale=true。
projection 缺失时返回 stale/minimal，不得现场扫描 logs/transcript/routes 补答案。
任何 summary / mcp / tools 模块不得调用 update_session_projection_from_event()。
```

- P0 不做完整 actor，但新增过渡 state file：

```text

```

该文件只是 v5.0 的过渡性 idempotency index，不是 durable command registry。它只能用于拒绝重复写入和缺字段写入，不能作为长期 command 事实源。v5.1 必须迁到 append-only `commands.ndjson + commands.index.json`，避免把单 JSON rewrite 当成长期命令事实源。

- 新增 helper：

```rust
fn check_or_record_idempotency(
    state: &AppState,
    owner: &str,
    session: &str,
    key: &str,
    command_fingerprint: &str,
) -> Result<IdempotencyDecision, String>
```

- 没有 `idempotency_key`：
  - read-only command 可选。
  - write command 一律返回 `rejected_missing_idempotency_key`。
  - destructive command 一律要求 `idempotency_key + precondition`，否则返回 `rejected_missing_safety_fields`。
  - bridge / daemon 不得在缺 key 时自动生成 key 后继续执行；调用方必须显式承担重试语义。
- `precondition.projection_last_session_seq` 不匹配：
  - 返回 `rejected_precondition`。
- 跨 session/daemon debug 才使用 `projection_last_global_seq`。

## Acceptance Criteria

- `cargo test -p agentcall-mcp -p agentcall-daemon` 通过。
- `cargo test --workspace` 通过或明确记录非本次失败。
- `agentcall_board(view=compact, filter=attention)` 返回不包含 `recent_events`、`legacy_detached_sessions`、raw logs、transcript 内容。
- `agentcall_session` 默认不包含 clean tail 和 plan excerpt。
- MCP timeout 不会拖满 Codex 120s。
- MCP timing log 能显示 daemon/time/size/truncated。
- MCP stdout 只包含合法 JSON-RPC response。
- write command 无 idempotency key 一律拒绝。
- destructive command 无 precondition 一律拒绝。
- tool response 默认 compact JSON 且有 cap；pretty/debug 只能走显式 debug 路径。
- Projection 只能 reducer 更新；summary/mcp 默认路径只能读。
- TUI parser 只能产生 hint，不得覆盖 hook-derived authoritative state。
