# AgentCall v4.0：Plugin-Provided MCP

## 背景

裸 MCP 配置依赖 `~/.codex/config.toml` 或项目 `.codex/config.toml`。在 Codex Desktop / background thread 场景中，我们观察到两个问题：

- 不同 session 读取的 `CODEX_HOME` 可能不同。
- MCP server 即使在某个配置文件中存在，也不一定注入到新线程的工具面。

v4.0 的目标是把 AgentCall 从“裸 MCP 配置”升级为 Codex plugin-provided MCP，让 MCP server 和使用说明一起作为插件能力安装。

## 结构

```text
plugins/agentcall/
  .codex-plugin/plugin.json
  .mcp.json
  skills/agentcall/SKILL.md

.agents/plugins/marketplace.json
```

`plugin.json` 负责声明插件元数据和 MCP companion file：

```json
{
  "name": "agentcall",
  "version": "4.0.0",
  "mcpServers": "./.mcp.json",
  "skills": "./skills/"
}
```

`.mcp.json` 暴露 AgentCall MCP：

```json
{
  "mcpServers": {
    "agentcall": {
      "enabled": true,
      "command": "E:\\Project\\AgentCall\\target\\debug\\agentcall-mcp.exe",
      "args": ["--workspace", "E:\\Project\\AgentCall"],
      "startup_timeout_sec": 120,
      "tool_timeout_sec": 120
    }
  }
}
```

## 安装

先构建 MCP 二进制：

```powershell
cargo build -p agentcall-mcp
```

注册本地 marketplace：

```powershell
codex plugin marketplace add E:\Project\AgentCall
```

安装插件：

```powershell
codex plugin add agentcall@personal
```

安装后必须完整重启 Codex Desktop，再新建线程。当前观察到：`codex plugin list` 可以显示 `installed, enabled`，但运行中的 Desktop host 不一定刷新 plugin registry，因此“安装后立刻新建线程”仍可能看不到 AgentCall。

安装成功后，`codex plugin list` 应显示：

```text
agentcall@personal  installed, enabled  4.0.0
```

## 验收

新开 Codex thread 后，执行最小检查：

```text
tool_search: agentcall
```

期望可见：

- `agentcall_daemon`
- `agentcall_board`
- `agentcall_route`
- `agentcall_session`
- `agentcall_session_send`
- `agentcall_report`

如果仍不可见，检查：

- `codex plugin list` 是否显示 `agentcall@personal installed, enabled`
- `codex plugin marketplace list` 是否显示 `E:\Project\AgentCall`
- 当前线程是否是安装插件之后新开的
- Codex Desktop 是否已经完整重启，而不是只新建线程

## 非目标

- v4.0 不恢复 ACP 主线。
- v4.0 不通过 HTTP fallback 绕过 MCP。
- v4.0 不把 AgentCall 写成每个项目都要复制一份的 `.codex/config.toml`。
- v4.0 不要求 MCP server 单实例；stdio MCP server 必须允许每个 Codex thread 各自启动。

## 决策

- daemon 仍然单实例。
- MCP server 作为薄 stdio bridge，允许多实例。
- 插件负责提供 MCP 声明和 Codex 使用说明。
- AgentCall 的跨项目能力依旧由 daemon `config/agentcall.local.json` 中的 `claude_workspace` 管理。
