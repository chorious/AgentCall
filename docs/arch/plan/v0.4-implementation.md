# AgentCall v0.4 实现说明

v0.4 把路线图里的双运行时编排落成了一个可测试的后端/MCP 版本：

```text
ACP                  = 短生命周期 agents-as-tools
Claude Code sessions = 长生命周期 handoff sessions
MCP                  = Codex/外部进程可调用的控制面
hooks                = Claude Code 事件进入 AgentCall 的入口
state                = `.agentcall/state/` 里的共享事实
```

## 已实现能力

### Context Packet

每次 bounded child call 会在调用前落盘：

```text
.agentcall/tasks/<task_id>/calls/<call_id>/input.json
.agentcall/tasks/<task_id>/calls/<call_id>/prompt.md
.agentcall/tasks/<task_id>/calls/<call_id>/context.json
```

这样可以审计两件事：

- child 最后做了什么：看 report
- child 当时知道什么：看 context packet 和 prompt

`ChildReport` 也加入了 `context_sufficiency` 字段，用来表达：

```json
{
  "status": "enough_to_act",
  "missing": [],
  "can_parent_resolve": true,
  "recommended_parent_action": ""
}
```

如果 child 返回 `need_context`，parent validation 会拒绝验收；当缺口无法由
parent 自动补齐时，结果会明确标记为需要 human clarification。

### Router

`agentcall route` 会根据任务特征给出运行时建议：

```powershell
python -m agentcall route "Review a focused diff for risk"
python -m agentcall route "Large exploratory migration across many modules" --needs-continuity
```

返回值是结构化 JSON：

```json
{
  "recommended_runtime": "acp",
  "expected_output": "ChildReport"
}
```

或：

```json
{
  "recommended_runtime": "claude-code-session",
  "expected_output": "CheckpointReport",
  "checkpoint_policy": "on stop, on idle, and after file edits"
}
```

### Hook Ingestion

Claude Code hook payload 可以进入 AgentCall：

```powershell
python -m agentcall hook ingest SessionStart --payload-json '{"session_id":"sess-1","agent":"Claude Code","pid":1234}'
```

AgentCall 会：

- 写入 `hook.<event>` 事件
- 更新 `.agentcall/state/active_sessions.json`
- 在 `SessionStart` / `UserPromptSubmit` 时返回可注入上下文

### Shared State 和 Board

v0.4 新增：

```text
.agentcall/state/
  project.json
  file_claims.json
  active_sessions.json
  context_index.json
```

统一状态面：

```powershell
python -m agentcall board --json
```

它返回：

- workspace
- tasks
- active_sessions
- recent_events
- reports
- project_state

### Checkpoint Request

长生命周期 handoff session 可以被主程要求 checkpoint：

```powershell
python -m agentcall checkpoint request sess-1
```

这会把 session 状态标为 `checkpoint_requested`，并写入事件。

## 测试证据

已跑过：

```powershell
$env:PYTHONPATH='src'
python -m pytest -q -p no:cacheprovider --basetemp .pytest_tmp_v04_tests
cargo test -p agentcall-mcp
cargo test -p agentcall-daemon
```

结果：

```text
Python: 11 passed
agentcall-mcp: 2 passed
agentcall-daemon: 0 tests, binary test target passed
```

MCP stdio smoke 使用独立 target dir 构建，验证了：

- `initialize` 返回 `agentcall-mcp` `0.4.0`
- `tools/list` 暴露 v0.4 工具
- `agentcall_route_task` 返回 `claude-code-session`
- `agentcall_context_packet_create` 落盘 context packet
- `agentcall_hook_ingest` 写入 session state
- `agentcall_session_spawn/status/tail` 启动瞬时 PTY 会话，最终 `stopped`，tail 返回 `AGENTCALL_SESSION_OK`
- `agentcall_checkpoint_request` 标记 checkpoint
- `agentcall_board` 返回统一 board state

## 当前边界

v0.4 是后端/MCP 可测试版本，还不是完整前端版本。

已完成：

- Codex 可以通过 MCP 查询 route。
- Codex 可以创建 context packet。
- ACP bounded workflow 会落盘 child input。
- Claude Code hook payload 可以进入 shared state。
- Codex 可以查询 session/report/event board。
- Codex 可以请求 checkpoint。

后续增强：

- 真实 Claude Code hook 安装脚本
- 更强的 file claim / conflict policy
- 状态 board 前端
- transcript 索引和摘要
- 更强 router 策略
