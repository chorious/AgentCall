# AgentCall v1.0 Release Notes

AgentCall v1.0 proves the first SOP layer and ships the first usable terminal
orchestration surface.

## Included

- Python SOP CLI for task creation, worker registry, runs, reports, events, and feedback.
- Rust PTY daemon with named sessions.
- Browser xterm.js UI with four-pane terminal supervision.
- WebSocket transport for PTY output, input, terminal responses, and resize events.
- File-based collaboration contract under `.agentcall/`.
- Documentation for SOP, session supervision, AgentAPI experiments, and Rust daemon direction.

## Proven

- Four Claude Code workers were supervised in one workspace.
- Four simple inspection tasks completed and produced reports.
- Four role-diverse second-layer planning tasks completed and produced separate artifacts.
- The orchestra was able to collect reports and accept work without source-file conflicts.

## Known Gaps

- Rust daemon sessions are still in-memory and are lost on daemon restart.
- WebSocket implementation is intentionally minimal and needs production hardening.
- Pane lifecycle can still be improved for session churn, focus retention, and reconnects.
- Claude prompt submission needs a first-class dispatch action rather than raw keystrokes.
- Runtime artifacts under `.agentcall/` are not committed by default.

## v1.0 Semantics

- `task.md`: written by the orchestra.
- `report.md`: written by the worker.
- `review.md`: written only for feedback, requested changes, blockers, or disputes.
- Clean acceptance is represented by task status or event metadata, not a ceremonial review file.
