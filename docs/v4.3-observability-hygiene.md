# AgentCall v4.3：Recent-first Logs + Stale Session Cleanup

## Summary

v4.3 解决近期调用日志暴露出的核心问题：board 查询被大日志拖慢、`active_sessions.json` 混入大量历史/unbound session、旧 queued 指令长期挂起，以及 Codex 轮询节奏过急。

本轮把 AgentCall 的运行数据拆成三层：

- **hot recent log**：board/session 默认读取，只保留最近窗口。
- **rotated archive**：按类型和大小归档，用于排障，不参与默认查询。
- **artifact payload**：大 stdout、Read 内容、PostToolBatch 结果外置，event 只保留摘要、大小、hash 和路径。

## Key Decisions

- `events.ndjson` 不再作为默认热查询源；新事件写入 `.agentcall/events/recent.ndjson`。
- hook 分类日志写入 `.agentcall/logs/hooks/<HookEvent>/recent.ndjson`。
- 单个 recent log 超过大小上限后归档到 `archive/YYYY-MM-DD/`。
- `PostToolUse` / `PostToolBatch` 中超过 4KB 的大字段写入 `.agentcall/artifacts/hooks/...`。
- 5 分钟无新消息、且不再由 live daemon 拥有的 historical/unbound session 会从 live 投影清理。
- 孤儿 `pending_supervisor_instructions` 会随 stale session 清理。
- Codex-facing patience 统一为 60 秒；stall 阈值为 300 秒。

## Non-goals

- 不引入 SQLite 或外部日志系统。
- 不删除旧 legacy `events.ndjson`；它作为历史兼容文件保留。
- 不自动 kill 仍由 daemon 拥有的 PTY worker。
- 不把 artifact 内容默认塞回 board/session summary。
- 不让 board 读取完整历史 archive。

## Operational Commands

```powershell
python agentcall.py logs doctor
python agentcall.py sessions cleanup --stale-after 5m
python agentcall.py sessions cleanup --stale-after 5m --apply
```

`sessions cleanup` 默认 dry-run。daemon 运行时也会在 board/health 路径上清理 stale live 投影。

## Acceptance Criteria

- `agentcall_board(view=compact, filter=attention)` 不再扫描 legacy 大日志。
- 大 `PostToolUse` / `PostToolBatch` payload 被 artifact 化。
- 5 分钟以上无更新的 unbound/Codex historical session 不再长期污染 live active session 计数。
- queued supervisor instruction 不再无限期挂在已失联 wrapper 上。
- `runtime_health` 暴露 `recent_first_logs=true`、`default_sleep_seconds=60`、`stale_session_ttl_seconds=300`。
