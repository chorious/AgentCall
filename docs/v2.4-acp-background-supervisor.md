# AgentCall v2.4: ACP Background Supervisor

v2.4 把 ACP 明确定义为后台 SOP worker：它可以比 MCP tool call 本身跑得久，但必须被 daemon 拥有、被权限策略约束、被 heartbeat 观测，并且有默认 hard timeout。

## 设计结论

- ACP 不是自由 runtime，也不是复杂任务的默认入口。
- ACP 只服务 v2.2 定义的 5 个 SOP：`read-and-report`、`evidence-check`、`contract-check`、`diff-review`、`single-report-update`。
- `agentcall_route(runtime=acp)` 快速返回 `route_id` / `invocation_id`，后续状态从 board、route 和 `acp_invocations.json` 读取。
- 默认 hard timeout 是 30 分钟，也就是 `1800s`。
- daemon 重启后不会宣称 ACP 可恢复；失去 ownership 的 running invocation 会标为 `orphaned_after_daemon_restart`，需要重新派发。

## Supervisor State

daemon 单写：

```text
.agentcall/state/acp_invocations.json
```

每个 invocation 记录：

- route / invocation / task / call identity
- workspace 与实际 child cwd
- template 与 report_path
- hard timeout、checkpoint threshold、heartbeat interval
- pid、heartbeat、last progress、permission denials
- report contract status
- running / checkpoint_due / completed / failed / failed_timeout / orphaned 状态

`routes.json` 只保留 compact projection，不再写入完整 ACP raw updates。

## Timeout / Checkpoint

- `acp_default_timeout_seconds`：默认 hard budget，默认 `1800`。
- `acp_max_timeout_seconds`：请求可用的最大 hard budget，默认 `1800`。
- `acp_checkpoint_due_seconds`：无进展多久后标记 `checkpoint_due`，默认 `600`。
- `acp_heartbeat_interval_seconds`：heartbeat 覆盖写 state 的间隔，默认 `60`。

`checkpoint_due` 是软信号，不会 kill ACP child。它表示这个 SOP 可能不适合 ACP，主管可以检查 board/report 后决定是否改派 PTY。

hard timeout 是硬边界。到期后 daemon kill 自己启动的 ACP child，并把 invocation 标为 `failed_timeout`。

## Concurrency Cap

默认同时最多运行 2 个 ACP invocation：

```json
{
  "acp_max_active_invocations": 2
}
```

超过上限时 route 返回 `acp_capacity_exceeded`。AgentCall 不排队，也不自动转 PTY，避免主管误以为任务已经开始。

## Board / Health

`GET /api/runtime/health` 增加：

- `acp_supervisor`
- `acp_active_invocations`
- `acp_max_active_invocations`
- `acp_default_timeout_seconds`
- `acp_max_timeout_seconds`
- `acp_checkpoint_due_seconds`
- `acp_heartbeat_interval_seconds`
- `acp_orphaned_invocations`

`agentcall_board(filter=attention)` 会展示 ACP 的 `checkpoint_due`、`failed_timeout`、`failed_report_contract`、`orphaned_after_daemon_restart` 和 route 级 `acp_capacity_exceeded`。

## 使用边界

ACP 首版仍只允许最多写一个 `report_path`。如果任务需要多文件实现修改、持续讨论、计划审查或人类可视化 handoff，应使用 PTY plan mode。
