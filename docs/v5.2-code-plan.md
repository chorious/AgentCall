# AgentCall v5.2 Code Plan：Durable Runtime + Scheduler + Adapter Trait

## Summary

v5.2 解决 “多 worker runtime 可恢复、可扩展、可审计”。

v5.0 定 contract 和 projection fast path；v5.1 做 SessionActor 和 PTY ownership；v5.2 再引入 durable store、worker scheduler、runtime adapter trait 和 confidence ledger。

一句话：

```text
v5.2 让多个 Codex-owned worker 可以被可靠调度、恢复和验收。
```

## Non-goals

- 不引入 Kafka/Redis。
- 不把 raw PTY logs / transcripts / binary artifacts 全塞进 SQLite。
- 不让 SDK runtime 替代 PTY default。
- 不破坏 v5.0 冻结的 MCP tool names。
- 不把 scheduler 做成分布式系统。
- 不让 worker 自动越权写同 workspace。
- 不实现隐式排队；没有显式 `allow_queue=true` 就必须直接拒绝。
- 不允许 AgentRuntime 绕过 SessionActor 直接写 PTY。
- 不让 ConfidenceLedger v1 依赖 LLM 解析自然语言 report。

## Hard Corrections From GPT Pro

v5.2 必须先做 durable invariants，再做 SDK runtime：

1. RuntimeStore trait 必须表达事务边界，不能把 event/projection/command/lease 拆成互相不一致的独立写。
2. SQLite schema 必须修正 `global_seq/session_seq` 和 nullable owner/idempotency scope。
3. SQLite 写并发必须明确：优先 StoreWriter actor，避免多个 actor 直接抢 DB writer。
4. WorkerScheduler v5.2.0 不实现隐式队列；capacity/workspace 冲突直接 reject。
5. WorkspaceLease 必须使用 canonical workspace key。
6. AgentRuntime 只能 submit command 到 actor，不能暴露第二条控制通道。
7. ConfidenceLedger v1 先 deterministic，不依赖 LLM report parsing。
8. SDK runtime 延后到最后，且必须 gated。

## Implementation Gate

v5.2 不以“加了 SQLite / runtime trait”为验收，而以 durable invariant 是否闭合为验收。任何 PR 合并前必须满足：

```text
1. daemon 主写路径只调用 transaction-shaped RuntimeStore API。
2. event append + projection update 必须同事务。
3. command accepted/completed + idempotency/projection/event 必须同事务。
4. route session + owner lease + workspace lease 必须同事务。
5. SQLite owner/idempotency scope 不允许 NULL 绕过唯一约束。
6. StoreWriter 是默认写并发边界；actor/route/hook/report validator 不直接抢 DB writer。
7. scheduler 不实现隐式 queue；没有 allow_queue=true 就只能 StartNow 或 Reject。
8. AgentRuntime 只能 submit command 到 actor，不能暴露 raw writer 或第二条 stop/interrupt/kill 通道。
```

SDK runtime、ConfidenceLedger、skill/context generator 都排在 durable store 和 scheduler 之后。若前置事务/租约/调度边界没有闭合，不允许为了“多 runtime 好看”提前推进 SDK runtime。

## Store Strategy

GPT Pro 建议 v5.2 引入 store 是合理的，但边界必须清楚：

```text
SQLite / indexed store:
  sessions
  runs
  commands
  event index
  projections
  reports index
  artifacts index
  idempotency keys
  owner leases
  workspace leases
  file tracker metadata

Filesystem:
  raw PTY logs
  clean tails
  Claude transcripts
  large reports
  diffs
  hook artifacts
  binary artifacts
```

Rust 依赖建议：

```toml
rusqlite = { version = "...", features = ["bundled"] }
```

但 v5.2 PR 1 先定义 trait，可先保留 JSON/NDJSON 实现；PR 2 先加入 StoreWriter actor 明确写事务边界，SQLite 在 PR 3 引入。

关键订正：`JsonRuntimeStore` 是兼容实现和迁移桥，不是新增一套长期 primitive write 路线。PR 1 可以先落 JSON store，但 daemon 主路径只允许调用 transaction-shaped API；任何 `append_event()`、`update_projection()`、`write_command_index()` 这类低层 helper 都必须保持 implementation-private 或 `#[cfg(test)]`。

## RuntimeStore Trait

`store.rs`：

```rust
pub(crate) trait RuntimeStore: Send + Sync {
    fn get_events(&self, query: EventQuery) -> Result<Vec<EventEnvelopeV1>, String>;

    fn get_session_projection(&self, session_id: &str) -> Result<Option<SessionProjectionV1>, String>;
    fn list_board_projection(&self, query: BoardQuery) -> Result<BoardProjectionV1, String>;

    fn get_idempotency(&self, owner: &str, key: &str) -> Result<Option<CommandRecord>, String>;

    fn save_report_index(&self, report: &ReportIndexRecord) -> Result<(), String>;
    fn save_artifact_index(&self, artifact: &ArtifactIndexRecord) -> Result<(), String>;

    fn renew_owner_lease(&self, lease_id: &str) -> Result<(), String>;

    fn record_file_read(&self, session_id: &str, path: &str) -> Result<(), String>;
    fn record_file_write(&self, session_id: &str, path: &str) -> Result<(), String>;

    fn append_event_and_update_projection(
        &self,
        event: &EventEnvelopeV1,
        projection_update: ProjectionUpdate,
    ) -> Result<AppendResult, String>;

    fn register_command_idempotently(
        &self,
        command: &CommandEnvelopeV1,
    ) -> Result<IdempotencyDecision, String>;

    fn complete_command_with_event(
        &self,
        command_id: &str,
        status: CommandStatus,
        event: &EventEnvelopeV1,
        projection_update: ProjectionUpdate,
    ) -> Result<(), String>;

    fn acquire_route_leases_and_create_session(
        &self,
        session: &SessionRecord,
        owner_lease: &OwnerLease,
        workspace_lease: Option<&WorkspaceLease>,
    ) -> Result<RouteDecision, String>;
}
```

事务边界要求：

```text
event append + projection update 必须同事务。
command accepted + idempotency index 必须同事务。
command completed + event append + projection update 必须同事务。
route session + owner lease + workspace lease 必须同事务。
```

写路径硬规则：

```text
RuntimeStore trait 暴露给 daemon 主路径的写操作必须是 transaction-shaped API。
append_event / update_projection / register_command / acquire_workspace_lease 这类 primitive write 不进入 public trait。
若测试、migration 或具体 store implementation 需要 primitive helper，只能作为私有方法或 #[cfg(test)] helper 存在。
```

StoreWriter 是 v5.2 的并发边界：

```text
所有 write transaction 进入 StoreWriter。
SessionActor / route / hook / report validator 不直接写 SQLite。
StoreWriter 返回结构化结果：committed / rejected / conflict / store_unavailable。
读路径可以走只读 connection 或 snapshot，但不得顺手 repair / writeback。
```

Implementations:

```text
JsonRuntimeStore
  wraps current .agentcall/state JSON + NDJSON paths.

SqliteRuntimeStore
  new durable indexed store.
```

SQLite 写并发策略：

```text
v5.2 默认采用 StoreWriter actor。
所有写 transaction request 进入单 DB writer 线程。

```

## WorkerScheduler

`scheduler.rs`：

```rust
pub(crate) struct WorkerScheduler {
    pub global_max_sessions: usize,
    pub per_owner_max_sessions: usize,
    pub per_workspace_policy: WorkspacePolicy,
    pub runtime_limits: HashMap<String, RuntimeLimit>,
}

pub(crate) enum ScheduleDecision {
    StartNow(StartSpec),
    RejectCapacity { reason: String },
    RejectWorkspaceLease { reason: String },
    RejectUnsupportedIsolation { reason: String },
}
```

禁止项：

```text
v5.2.0 不定义 Queue variant。
不返回 pending/queued 幻觉状态。
只有未来显式实现 allow_queue=true 时，才能新增 QueueExplicit。
```

P0 for scheduler:

- No hidden queue by default.
- If capacity exceeded, return `capacity_exceeded`.
- If workspace exclusive lease exists, return `workspace_busy`.
- Codex can request `worktree_isolated`, but v5.2 must actually create/use worktree or reject.
- Queue 不在 v5.2.0 默认实现中出现。
- 未来若实现 queue，必须显式请求 `allow_queue=true`，并返回 `QueueExplicit { queue_id, estimated_position }`。

Workspace key:

```text
WorkspaceLease 使用 canonical workspace key。
Windows 下必须处理大小写、斜杠、尾部点、\\?\ 前缀、symlink/junction/subst drive。
无法确认同一性的路径必须 reject 或 warning，不能默认为不同 workspace。
WorkspaceLease.workspace_key 必须来自 canonicalizer，而不是直接使用 route 传入的 workspace 字符串。
```

`agentcall_route` integrates:

```text
handle_route()
  -> scheduler.decide()
  -> AgentRuntime.start()
  -> store sessions/runs/leases
```

## AgentRuntime Trait

`runtime.rs`：

```rust
pub(crate) trait AgentRuntime: Send + Sync {
    fn id(&self) -> &'static str;
    fn capabilities(&self) -> RuntimeCapabilities;
    fn start(&self, spec: StartSpec) -> Result<RuntimeSession, String>;
    fn submit_command(&self, session_id: &str, command: CommandEnvelopeV1) -> Result<CommandDecision, String>;
    fn observe_events(&self, session_id: &str, cursor: EventCursor) -> Result<Vec<EventEnvelopeV1>, String>;
    fn collect_artifacts(&self, session: &RuntimeSession) -> Result<Vec<ArtifactIndexRecord>, String>;
}
```

规则：

```text
ClaudeCodePtyRuntime::submit_command(...)
  -> actor_registry.submit(session_id, command)

interrupt / stop / kill 都是 CommandType。
AgentRuntime 不允许拿 raw writer，也不提供第二条控制通道。
SDK runtime 也必须映射到同一 EventEnvelope / Projection contract。
`submit_command` 是唯一控制入口；`interrupt` / `stop` / `kill` 不能作为 runtime trait 的独立方法重新暴露。
```

Implementations:

```text
ClaudeCodePtyRuntime
  default stable runtime.
  wraps SessionActor + ProcessController.

ClaudeCodeSdkRuntime
  experimental.
  no default routing.
  only behind explicit config/capability.
```

Do not reintroduce ACP as default.

ConfidenceLedger v1 只做 deterministic rules：

```text
high/medium evidence:
  hook-derived file write
  command event
  explicit test result event
  policy block

  Claude report natural-language claim
  raw terminal text
  inferred test pass
```

第一版不依赖 LLM 解析 report 自然语言。可以要求 Claude report 输出结构化 JSON claims；没有结构化 claims 时只作为低可信 claim。

## Acceptance Criteria

- daemon restart preserves projections/events/idempotency keys.
- `agentcall_board` with 10 sessions stays projection/index-only.
- `agentcall_board(scope=mine)` filters by owner.
- workspace exclusive lease prevents default concurrent writes.
- event cursor query works.
- event append + projection update are transactional.
- SQLite owner/idempotency scope is non-null and cursor-safe.
- scheduler does not return hidden queue.
- ConfidenceLedger detects claim/artifact/test mismatch.
- PTY runtime and SDK runtime share same upper contract.
- Raw logs/artifacts remain filesystem-backed.
