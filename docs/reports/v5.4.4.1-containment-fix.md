# AgentCall v5.4.4.1 Containment Fix

## Summary

This patch fixes the containment regression exposed by the RKV parallel workers:
read-only shell probes and absolute target-workspace paths were too easy to
misclassify as unsafe while Claude Code was running from the configured
`claude_workspace` (`D:\guKimi`).

## What Happened

- Worker process cwd was correctly forced to `D:\guKimi`.
- Route `workspace` correctly pointed at `E:\GameProject\RKV`.
- Workers attempted to inspect or create output under
  `E:\GameProject\RKV\.agentcall-parallel`.
- A read-only Bash probe like:

```text
ls -la "E:\GameProject\RKV\.agentcall-parallel\vic2" 2>/dev/null || echo "Directory does not exist yet"
```

was denied as non-read-only because the Bash classifier treated all `>` and
all `echo` usage as write-like.

## Fix

- Allow stderr-to-null redirects in Bash read-only probes:
  - `2>/dev/null`
  - `2> /dev/null`
  - `2>nul`
  - `2> nul`
  - `2>$null`
  - `2> $null`
- Keep denying actual file writes through shell redirection.
- Preserve denial for creation/deletion/copy/move commands.
- Add regression coverage for absolute writes inside a route target workspace
  materialized through `writable_roots`.

## Validation

```text
cargo test -p agentcall-daemon
```

Result: `143 passed`.

## Not Covered

This patch does not address the separate route prompt timing issue where
`pty.input_sent` can happen before Claude Code emits `SessionStart`.
