# v6.9.2 PR Report: Current Board, Workspace Contract, Flow Skill

Date: 2026-06-27

## Summary

v6.9.2 closes the compact-board historical projection regression and starts the contract cleanup requested for worker permissions and flow maintenance.

## Changes

- Compact board now filters store projection rows through the daemon current live-session index. Historical `needs_attention` rows are excluded from ordinary compact board payloads and remain debug/raw material.
- PTY containment now emits `workspace_contract.v1` with task root, runtime root, artifact root, writable/readonly roots, deny-write roots, and Bash side-effect mode.
- Report/review workers now use created-artifact semantics in hook pre-policy: new report/scratch artifacts are allowed, later edits require same-session claims, and pre-existing project files are denied.
- Report/review Bash is readonly in pre-policy. Coding/edit Bash remains monitored by folder heartbeat plus destructive-command preflight.
- Real Claude coding routes now require a linked git worktree on a non-main branch and surface `coding_requires_worktree`, `worktree_path`, `branch`, `merge_requires_pr`, and `pr_report_path`.
- `agentcall-flow` is repository-managed from `docs/agentcall-flow/` and generated into `.codex/skills/agentcall-flow`.
- `runtime-release` now requires an explicit skill update decision via prompt, `--update-skills`, or `--skip-skill-update`.

## AgentCall Flow

AgentCall workers were not launched for this release because the active failure mode was compact board pollution itself. The local implementation followed the repository-managed phase gates and records this report as the closure artifact.
