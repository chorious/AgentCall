# v6.9.2 Plan: Current Board, Worker Permissions, Flow Skill

Date: 2026-06-26

## Goal

Ship v6.9.2 as one coherent control-plane cleanup:

1. Compact board must not mix historical projections into current worker state.
2. Worker permissions must match worker kind:
   - `report` workers may create and then edit their own report/scratch artifacts, but must not modify existing project files.
   - `coding` workers must use an explicit worktree/branch/PR-style workflow instead of writing the canonical workspace directly.
3. `agentcall-flow` must be maintained by this repository and checked during release/update flows.

## Out Of Scope

- Do not rewrite the frozen `docs/v6.2-code-plan.md`.
- Do not revive ACP/SDK as the main runtime.
- Do not make Python a live state writer.

## Work Items

### A. Board Current Index

- Change compact board filtering so ordinary `view=compact` returns only current live daemon sessions.
- Keep historical projections for debug/raw surfaces only.
- Add response flags showing compact board is current-indexed and historical projections were excluded.
- Add tests with many stale/historical projections and a small live set.
- Preserve read-path discipline: no cleanup, no worker snapshot recomputation, no raw terminal scan.

### B. Report Worker Created-file Policy

- Add session-scoped created-file ownership checks for report workers.
- Allow report workers to create their own report/scratch artifacts and continue editing files they created.
- Reject report workers modifying pre-existing project files with structured policy details.
- Keep report worker Bash behavior conservative when a write cannot be attributed to created report/scratch files.

### C. Coding Worktree / PR Gate

- Require coding workers to run in an explicit worktree path and branch.
- Reject coding routes targeting the canonical workspace directly.
- Return clear route/session/board projection fields:
  - `coding_requires_worktree`
  - `worktree_path`
  - `branch`
  - `merge_requires_pr`
  - `pr_report_path`
- Keep report workers on shared report lease behavior.

### D. agentcall-flow Repository Maintenance

- Import `agentcall-flow` source into the repository.
- Generate/check `.codex/skills/agentcall-flow/SKILL.md` from repo sources.
- Extend release-check to validate both supervisor and flow skills.
- Extend runtime-release/update script to require an explicit skill-update decision:
  - interactive prompt, or
  - non-interactive `--update-skills` / `--skip-skill-update`.

### E. Version And Docs

- Bump public product version to `6.9.2` across README/CHANGELOG, crates, pyproject, plugin manifest, MCP server version, and Cargo.lock.
- Update AGENTS, README, docs index, and implementation report.

## Validation

- `cargo fmt`
- `cargo test --workspace`
- `python -m pytest -q`
- `python scripts\agentcall_arch_audit.py`
- `python agentcall.py release-check`
- `python agentcall.py runtime-release --version 6.9.2`

## AgentCall Flow Note

This task is intentionally implemented locally first because compact board pollution is the active failure mode and adding more AgentCall workers would worsen the control-plane signal. The repository-managed `agentcall-flow` skill is part of this release so future large changes can safely use the flow.
