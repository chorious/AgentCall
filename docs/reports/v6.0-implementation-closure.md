# AgentCall v6.0 Implementation Closure

日期：2026-06-11

## Summary

v6.0 按冻结计划 `docs/v6.0-code-plan.md` 推进，计划文件未修改。当前实现完成了 PTY-first worker 的短事务路由、prompt gate、projection-first worker state、report-ready 验收闭环，以及 MCP 默认返回面的压缩。

本轮发现并修复了一个真实并发 smoke 暴露的问题：`agentcall_report(action=accept, session_id=...)` 原先把相对 `report_path` 解析到 daemon workspace，导致 worker 在 route target workspace 中实际写出报告后仍被判定为 `report_missing_or_empty`。现在验收会按 route `workspace` 解析相对报告路径，并在返回中暴露 `report_abs_path` 便于诊断。

## Implemented

- Route 默认走 PTY utility worker，不再依赖 ACP。
- Route start 快速返回，并生成 `.agentcall/tasks/<route_id>/prompt.md` handoff。
- Prompt gate 追踪 `prompt_pending`、`prompt_missing`、`UserPromptSubmit` ack 与 `submit_pending_prompt` 自愈动作。
- Worker state 投影包含 `starting`、`prompt_pending`、`prompt_missing`、`working`、`waiting_input`、`needs_permission`、`blocked_by_policy`、`report_ready`、`stopping`、`done`、`failed`。
- MCP canonical 工具面保持小集合，`agentcall_route`、`agentcall_session`、`agentcall_session_send`、`agentcall_report` 默认返回 compact projection。
- `agentcall_session_send` 隐藏 raw lease/precondition 细节，但 destructive action 仍由 daemon-owned control token 保护。
- Report accept 使用 deterministic evidence，支持 route target workspace 下的相对报告路径。
- `scripts/agentcall_dev.py daemon-health` / `doctor` 支持 daemon token 头，避免本机 token 开启后诊断脚本误报 401。

## Known Host-Level Issue

当前 Codex 线程内置的 `mcp__agentcall` transport 仍返回 `Transport closed`。这不是 daemon 运行问题：daemon HTTP `/api/mcp/call` 已用 token 成功完成 route/report smoke。该问题属于 Codex 宿主 transport 未重新绑定旧 MCP 进程，通常需要重启 Codex 线程或宿主侧重新加载 MCP。
