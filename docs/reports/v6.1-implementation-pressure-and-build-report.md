## Build / Smoke Problems Encountered

| Problem | Cause | Resolution |
|---|---|---|
| `cargo build --workspace` initially failed because daemon/MCP binaries were locked | Existing `agentcall-daemon.exe` and `agentcall-mcp.exe` processes held the target binaries | Stopped old processes, rebuilt successfully |
| Persistent daemon launched from this Codex shell did not survive command return | Codex host job cleanup terminates detached children started by `Start-Process`, `cmd start`, and Python detached process attempts | Used bounded temporary daemon inside smoke scripts for verification; live Codex MCP transport still requires host/plugin reconnect |
| `CREATE_BREAKAWAY_FROM_JOB` detached launch failed | Windows denied breakaway from the current job object | Treated as host constraint, not daemon code path |
| `schtasks.exe` launch route failed | Local scheduled-task invocation failed even for a trivial test program | Not used for v6.1 validation |
| Current Codex `mcp__agentcall.agentcall_daemon(status)` returns `Transport closed` | Codex host still points at a closed MCP transport | Documented as current host transport issue; daemon API smoke and `/api/mcp/call` are verified independently |
| Real worker smoke initially lacked daemon token | Smoke temp config had token auth enabled but requests did not send `X-AgentCall-Token` | Added smoke daemon token and request header |
| Smoke expected old raw route status shape | v6.1 route returns normalized worker summary | Updated smoke to assert v6.1 `schema_version=2`, `worker`, and worker `state` |
| Smoke depended on raw/clean PTY text markers | PTY clean output is not a reliable state contract | Updated smoke to validate route/session/report projection instead of raw terminal output |
| Report accept could be high from artifact-only evidence | v6.1 requires daemon-observed write for `overall=high` | Added hook write evidence in smoke and split confidence assertions |

## Current Remaining Operational Caveat

The implementation/build/test target is complete, but this Codex session's AgentCall MCP transport is still closed:

```text
agentcall/agentcall_daemon -> Transport closed
```

This is outside the temporary daemon smoke path. The smoke tests start their own daemon and call `/api/mcp/call` directly with token auth, proving the daemon/MCP request semantics. The live Codex tool transport still needs the host/plugin side to reconnect before this exact Codex session can call `agentcall_*` tools again.

## Public Old Status Audit

Runtime code no longer emits `pending_prompt_commit_sent`. Remaining occurrences are limited to:

- frozen plan text,

## Conclusion

v6.1 is functionally implemented and validated against the frozen plan's key smoke and pressure requirements. The only unresolved item is the current Codex host's closed MCP transport, which is a live tool-connection problem rather than a failing daemon/API/smoke path.
