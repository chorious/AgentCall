# AgentCall v5.2 Live Write Audit

Date: 2026-06-09

## Scope

This audit covers default live write paths for the v5.0-v5.2 runtime boundary. The goal is to prove that durable core state is written through `RuntimeStore` / `SessionActor` transaction boundaries, while remaining direct JSON writes are classified as legacy side-indexes, operator metadata, or test/debug fixtures.

## Durable Core Boundary

| State family | Default writer | Evidence | Status |
| --- | --- | --- | --- |
| Events + session projections | `state.rs` helpers calling `RuntimeStore::append_event_and_update_projection` | `append_agent_event_locked`, `complete_command_event`; architecture audit forbids direct calls from other modules | Closed |
| Command accepted/completed state | `commands.rs` prepares idempotent command, `actor.rs` completes via `complete_command_event` | command idempotency tests, actor command tests, JSON/SQLite rollback tests | Closed |
| Route session + owner/workspace leases | `routes.rs` reserves then commits via `RuntimeStore::acquire_route_leases_and_create_session` | route lease transaction rollback test, real-worker smoke JSON/SQLite | Closed |
| MCP board/session default read path | projection store only | compact attention board tests, MCP session projection tests, architecture audit | Closed |
| Owner-scoped board view | projection store filtered by owner | `compact_attention_board_can_filter_to_current_owner_projection` | Closed |

## Remaining Direct JSON Writers

| Path family | Writer | Why it remains | Default MCP live authority? | Follow-up |
| --- | --- | --- | --- | --- |
| `file_claims.json` | `hooks.rs` | legacy hook policy side-index used for permission decisions | No for projection/command/session; yes for current hook policy compatibility | Candidate for future `RuntimeStore` file-claim table |
| `active_sessions.json` | `hooks.rs`, `routes.rs`, stale cleanup | legacy compatibility for hook context and old viewers | No; live daemon sessions and projection are authoritative | Keep as compatibility until viewer fully projection-backed |
| `runtime_binding.json` | `hooks.rs` | hook-to-wrapper binding side-index | No for command/projection durability; yes for hook-aware summary hints | Candidate for future binding table |
| `pending_supervisor_instructions.json` | `hooks.rs`, `summary.rs` cleanup | delivery queue for hook additionalContext | No; it is a delivery side queue, not session truth | Keep; add expiry/cleanup tests as needed |
| `policy_denials.json` | `hooks.rs` | repeated-denial aggregation and corrective context | No; projection receives blocking events separately | Keep as side-index |
| route/context/transcript indexes | `routes.rs` | operator artifacts and report/context metadata | No for session projection; routes are separate control records | Future store migration if route recovery becomes strict |

## Architecture Audit Coverage

`scripts/agentcall_arch_audit.py` now checks:

- tracked runtime/build outputs are not committed,
- PTY raw writer ownership stays inside session startup/actor boundary,
- default MCP session uses `session_projection_summary`,
- compact attention board returns projection before cold reads,
- event/projection transaction store calls do not appear outside `state.rs` and store implementations.

## v5.2 Decision

For v5.2 completion, durable core invariants are considered closed when:

1. event/projection writes use `RuntimeStore` transactions,
2. command completion uses `RuntimeStore` transactions,
3. route session and lease creation use `RuntimeStore` transactions,
4. owner-scoped board uses projection-only reads,
5. remaining direct JSON writers are not treated as durable session truth.

The remaining direct JSON writers are intentionally documented as side-index debt, not as v5.2 durable core blockers.
